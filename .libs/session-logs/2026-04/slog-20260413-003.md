### 2. 段階 2 設計 + 実装

RSRC-WEBANIM-HARDCASE §1 に沿って WebGL 取材パイプラインを追加:

- **constants.py**: `WEBGL_OUTPUT_DIR_NAME = "webgl"` / `WEBGL_SHADERS_FILENAME = "shaders.json"` を追加
- **recon.mjs**: `collectReport` 内で `canvas.getContext('webgl'|'webgl2')` 走査 → `report.webgl = { hasWebGL, canvasCount }` を返す。`main()` の desktop viewport 処理で `hasWebGL=true` なら `spector.mjs` の `captureWebGLShaders(page)` を呼んで `shaders.json` に書き出す
- **spector.mjs 新規**: 当初 Spector.js CDN を `addScriptTag` で注入する設計だったが方針転換 (§3 参照)。最終形は WebGL 標準 API (`getAttachedShaders` + `getShaderSource`) で `CURRENT_PROGRAM` の shader 本文を直接取得する実装
- **clone.py**: `_format_webgl_section()` 追加 → 本棚ページに「WebGL / シェーダ取材」セクション生成。`_find_shaders_json()` + `_build_tsx_skeleton_r3f_shader()` 追加 → r3f 雛形を `shaderMaterial` + `uniforms` (u_time / u_resolution) 版に分岐
- **specs/03-tools.md**: 段階 2 組み込み内容 + `spector.mjs` ファイル名の歴史的経緯 + `CURRENT_PROGRAM` ベースの制約を追記

### 3. Spector.js → WebGL 標準 API への方針転換

最初の実装 (Spector.js を CDN から注入、`spyCanvases()` + `captureNextFrame()` で programs 抽出) で動作検証したところ:

1. 当初使った CDN URL `https://spectorcdn.babylonjs.com/spector.bundle.js` は 404 を返す (誤 URL)
2. `jsdelivr` に切り替えたら CDN 自体は通ったが、capture 結果の `programs` が常に空
3. デバッグ情報 (capture の構造 dump) を取ったところ、**Spector の capture には `compileShader` / `shaderSource` / `linkProgram` が一切含まれない** ことが判明。Spector は「1 フレームの描画 call」のみ記録し、初期化フェーズは対象外
4. 対策として WebGL 標準 API (`getAttachedShaders(program)` + `getShaderSource(shader)`) で `CURRENT_PROGRAM` から直接取る実装に全面書き換え → 即座に `threejs.org` の Three.js RawShaderMaterial (vertex 398 chars / fragment 5063 chars) 抽出成功

### 4. testgl での動作確認 (LAYOUT-TEST-WEBGL)

## SLOG-20260413-003.決定事項 — 決定事項

1. **初回リリースは `v1.0.0`** — タグ 1 件もなし + VERSION=1.0.0 の状態だったため、段階 1 完全動作検証済みセットを最初の公式リリースとして公開
2. **Spector.js を捨てて WebGL 標準 API 経由に切替** — Spector の capture 仕様 (1 フレーム call のみ) では GLSL 抽出不可と実測で判明。`getAttachedShaders` + `getShaderSource` で確実に取れることを実証。副産物として CDN 依存も消えてオフラインで動作可能に
3. **ファイル名は `spector.mjs` のまま残す** — 実装から Spector.js を消しても名称は歴史的経緯として残置。specs/03-tools.md にその経緯を明記
4. **GLSL は雛形に直接埋め込まない** — 本棚ページ + `shaders.json` から AI が貼り込む方針。長大な GLSL のエスケープ / 維持コストを避ける
5. **`CURRENT_PROGRAM` ベースの制約を受け入れる** — 描画ループの最終 program のみ抽出される。全 program 履歴が要るケースは段階 4 以降で `addInitScript` + WebGL hook 方式に拡張する前提で、段階 2 では単純実装を優先
6. **段階 2 の検証対象は Three.js 代表例で十分** — `threejs.org/examples/webgl_shaders_ocean.html` で GLSL 抽出 + 本棚ページ生成 + 雛形分岐まで通ったので、他の WebGL サイトでの追加検証は次セッション以降 (R3F 以外のライブラリ検出など)
