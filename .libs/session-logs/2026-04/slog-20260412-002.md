## SLOG-20260412-002.概要 — 概要

プル子のインストールを 1 コマンドで完結させる `bootstrap.py` + スターターテンプレートを作成。さらにレイアウトデザイン機能を /codi から分離して `/layo` 専用スキルを新設し、デザインリファレンス (`_documents/` → `docs/`) をシステムに接続した。

## SLOG-20260412-002.作業内容 — 作業内容

### 1. bootstrap.py 作成
- プロジェクトディレクトリで 1 コマンド実行するだけでプル子をインストールできるスクリプト
- スクリプト自身の場所からソースリポジトリを自動検出（ローカルコピー）、検出できなければ giget で GitHub 取得
- git init → .nxt-core コピー → install.py 実行を自動化
- Windows UTF-8 の line_buffering 問題を発見・修正

### 2. スターターテンプレート (`starter/`)
- Next.js 15 + Tailwind v4 + Storybook + Playwright の全部入りテンプレート
- package.json, next.config.ts, tsconfig.json, postcss.config.mjs, globals.css (@theme トークン), layout.tsx, page.tsx, .storybook/main.ts, .storybook/preview.ts
- bootstrap.py がテンプレートコピー → npm install → npx playwright install まで全自動
- constants.py DEFAULT_GITIGNORE_LINES に storybook-static, test-results, playwright-report を追加

### 3. /layo スキル新設 — デザイン設計の分離
- /codi ステップ 2-B の設計工程 (recon/dump/apply/rules) を /layo に分離
- /codi は実装工程 (build/assemble/baseline) のみに簡素化
- skills/layo/SKILL.md 作成、specs/01-workflow.md, 02-skills.md フロー図更新

### 4. layo.py オーケストレーター — スクリプト駆動化
- core/layo.py 新規作成: load.py と同じパターンのコンテキスト収集スクリプト
- モード判定、input.md 読み込み、キーワード抽出、docs 検索、コーディングルール参照、プロジェクトルール収集、関連ページ収集を機械的に実行
- load.py の _collect_rules → collect_rules に公開化して DRY (L1 §1.2)
- load.py からレイアウトタスクの docs 検索を削除 (/layo に移行)

### 5. デザインリファレンス統合 (`_documents/` → `docs/`)
- `_documents/` を `docs/` にリネーム (specs/, core/, skills/ と並ぶルート直下)
- 旧インデックス (_docs_index.json, INDEX.md) を全削除
- paths.py: DOCS_DIR を NXT_ROOT / "docs" に変更 (.libs/docs/ から移行)
- constants.py: LIBS_SHELVES から "docs" を削除
- bootstrap.py: DIST_DIRS に "docs" 追加 (プロジェクトにコピー対象)
- docs.py: コメントを更新
- Apple HIG, shadcn, Tailwind, M3, Motion, Three.js のリファレンスが検索可能に

### 6. 仕様書更新
- specs/01-workflow.md: フロー図に /layo 追加
- specs/02-skills.md: /layo 説明追加、スキル数 7 に更新
- specs/03-tools.md: layo.py, docs.py 更新、DOCS_DIR 追加、依存関係図更新
- specs/04-project-guide.md: 棚表から docs/ 削除、storybook 作成元を /layo に変更

## SLOG-20260412-002.決定事項 — 決定事項

1. **bootstrap.py はルート直下に配置** — release.sh と同じ位置。.gigetignore に追加して giget 配布から除外
2. **スターターテンプレート方式を採用** — create-next-app を動的実行する方式ではなく、テスト済みテンプレートを配布する方式。理由: 確実性重視、仕様変更リスク回避
3. **Playwright + Storybook は bootstrap.py で自動インストール** — ユーザーのインストール忘れを防ぐ。npm install + npx playwright install まで全自動
4. **/layo を独立スキルに分離** — /codi の 7 サブステップが複雑すぎた。設計 (recon/dump/apply/rules) と実装 (build/assemble/baseline) を分離。/layo は何度でも再実行可能な対話型デザインツール
5. **layo.py オーケストレーター必須** — specs/08-responsibility.md に従い、ワークフロー制御は Python スクリプトが行う。AI は内容を書くだけ
6. **`_documents/` → `docs/`** — デザインリファレンスを正式ディレクトリとしてシステムに接続。DOCS_DIR を NXT_ROOT / "docs" に変更
7. **load.py の collect_rules を公開化** — layo.py から import して DRY を維持。レイアウトタスクの docs 検索は /layo に完全移管
