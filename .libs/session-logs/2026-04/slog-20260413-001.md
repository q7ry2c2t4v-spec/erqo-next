## SLOG-20260413-001.概要 — 概要

任意のウェブサイトのアニメーション/エフェクトを AI が忠実に取材・再現するためのリサーチと実装を一気に進めた。RSRC-WEBANIM-{CAPTURE, REPLAY, HARDCASE} の 3 ページで取材・再現・難所攻略の設計書を作り、Plan モードで段階 1〜5 に分割した計画を承認。段階 1 (基本セット) を本セッションで `core/clone_node/recon.mjs` / `core/clone.py` / 新規 `diff.mjs` / `record.py` に組み込んだ。既存の `RSRC-WEBCLONE-RULES §4.2` の framer motion TODO は完全重複していたため削除し、WEBANIM-REPLAY へ参照移譲した。

## SLOG-20260413-001.決定事項 — 決定事項

1. **忠実性最優先 — 中途半端な再現を禁止** — ユーザー要件として「元サイトと並べて区別がつかない」水準を目指す。「だいたい動く」は不採用
2. **段階投入方式を採用** — 段階 1 は npm 追加依存なしの基本セット、段階 2 以降は別セッションで段階的に。1 セッションで全部詰めると品質が下がるため
3. **元サイトと同じライブラリを使う原則** — GSAP サイトを Motion で代用しない。検出結果に応じて雛形を出し分ける
4. **SSOT は JSON ファイル** — `anim_lib_patterns.json` を両言語 (Python / Node) から読むことで L1 §1.2 DRY を言語またぎで維持
5. **WEBCLONE-RULES §4.2 は完全削除 → 参照移譲** — WEBANIM-REPLAY と完全重複していたため整合性を優先
6. **差分検証は段階 1 ではレポートのみ** — 自動修正ループ (段階 5) までは手動確認前提。Vision LLM フォールバックも段階 4 まで保留
7. **baseline.mjs の reducedMotion は維持** — 元サイト vs 再現は「静止フレーム同士」で比較。動きの一致検証は段階 5 で扱う
8. **プロジェクト側依存追加は starter 経由** — 本元に node_modules は作らない。`starter/package.json` 経由で bootstrap.py がプル子に配布
