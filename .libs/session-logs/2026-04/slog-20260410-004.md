## SLOG-20260410-004.決定事項 — 決定事項

1. **PC 個人スコープ（`~/.claude/settings.json`）に erqo-next 関連の設定を書かない**
   - 理由: ユーザーは複数デバイスでこの OS を使う。PC ごとの設定は他デバイスと共有できず、
     挙動がバラバラになって混乱する
   - 例外: Claude Code 自身の純粋な個人嗜好（テーマ等）は OS と無関係なので OK

2. **共通スコープは「constants.py に出どころ + install.py/dev.py が配置」で実装**
   - 理由: 単一の出どころ（constants.py）があれば、変更は 1 か所で済む。配置先が複数（本元と
     プロジェクト各々）でも、内容は一致する
   - 副次効果: 既存プロジェクトも `python core/install.py --update` で自動取り込み可能

3. **古いコードは積極的に削除する**
   - 理由: 「将来使うかも」で残すと『消し忘れ』が増える。今使われていないものは消す
   - 削除した push/full は手動で `git push` `git commit && git push` を打てば代替できる

4. **specs/07-scopes.md を独立ファイルにした**
   - 理由: コーディング規約（specs/06）と設定スコープは関心事が違う。混ぜると検索性が落ちる
   - 今後ルールが増える時の起点としても使える
