## SLOG-20260413-007.決定事項 — 決定事項

### 1. スタック対応方式: インストーラー分岐を採用

- **理由**: 中身の大部分が Next.js 非依存のため、共通部分を 1 つの本元で進化させれば全スタックに届く
- **却下: ブランチ分岐**: 共通の改善を 2 本のブランチに都度反映する二重メンテが長期で重い

### 2. 呼称体系: エル子 / プル子 / プタ子

- エル子 = 本元（`IS_SOURCE=True`）
- プル子 = Next.js プロジェクト（既存動作維持）
- プタ子 = 任意スタックのプロジェクト（新規）

### 3. ディレクトリ構造

```
本元
├── 共通骨格（記録・セッション・git・監査・リサーチ 等）
│       → プル子・プタ子どちらにも配布
└── stacks/
    ├── nextjs/     ← プル子用（layo, clone, docs, L3, starter）
    └── generic/    ← プタ子用（最小セット、layo なし）
```

### 4. プル子 / プタ子の機械的区別

`.claude/state/` に variant の印を 1 個置いて、全スクリプト・全スキルがこの印を機械参照して動作を切り替える。AI の自己判断は挟まない（`specs/08-responsibility.md` 責務分離と整合）。

### 5. プタ子の仕様

- `/layo` は配布しない（Next.js 特化装備のため）
- スタック選択は最初は **インストール時の対話** で実施。独立スキル化（`/stack`）は必要になってから追加

### 6. 実装方針: 段階分け

1. 共通骨格の切り離し + 区別の印の導入（土台）
2. Next.js 装備を `stacks/nextjs/` に引っ越し（既存プル子動作維持）
3. `stacks/generic/` と `install.py` のプタ子分岐を追加
4. 必要になったら `/stack` スキル追加
