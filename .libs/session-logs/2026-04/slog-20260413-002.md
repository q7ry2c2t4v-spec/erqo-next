### 4. バグ発見 → 本元修正

**症状**: input.md の採用ライブラリセクションを空欄にしたのに、`build` が `採用ライブラリ: gsap` を選んでしまう。本棚ページの採用ライブラリセクションは「検出なし」なのに矛盾。

**原因**: `_detect_adopted_library` (`core/clone.py:1630`) が input.md の採用ライブラリセクション全体を lowercase した文字列から `\b<library_name>\b` で正規表現検索する。テンプレの案内文 `- (空欄なら recon 検出結果から自動選択。指定するなら gsap / motion / lenis / r3f / lottie / pure-css のいずれか)` に含まれる `gsap` が最初にヒットするため、ユーザーが空欄にしても必ず GSAP が選ばれていた。

**修正** (本元 `core/clone.py`):
1. `_build_input_template` のテンプレから例示部分を HTML コメントに移動 (`<!-- 指定する場合は gsap / motion / ... -->`)
2. `_detect_adopted_library` 内で、箇条書き行の中身が `(` または `<!--` で始まるものをプレースホルダ扱いでスキップ
3. 修正後のファイルを `testgl/.nxt-core/core/clone.py` に単純コピー → build 再実行で pure-css が正しく選ばれることを確認

## SLOG-20260413-002.決定事項 — 決定事項

1. **段階 1 検証は testgl で bash 直叩き方式** — エル子セッション内から `/layo` `/codi` スキルは呼べないため、`clone.py <subcommand>` を直接叩いて 7 サブステップを通す方針を採用
2. **取材 URL は sodaterulab.jp** — ユーザー指定。JS アニメ非使用サイトだったため「検出ゼロ分岐」を検証できた。GSAP 雛形は `- gsap` 明示指定で別途検証したため、追加 URL での再検証は不要と判断
3. **バグ修正は 2 段構え** — テンプレ側の例示を HTML コメントに移動するだけでは既存ユーザーの input.md が救えないため、検出ロジック側でも括弧/HTML コメント行をスキップする二重防御を採用
4. **再配布は単純コピーで検証継続** — bootstrap 完全再実行 (npm install + Playwright DL で 10 分以上) は検証のたびに回すとコスト過大。単純コピーで代替して検証効率を優先
5. **diff 差分率 63%/46% でも段階 1 成功扱い** — 実装コード (.tsx) が雛形指示だけで未実装のため高差分率は想定内。段階 1 の検証対象は「パイプラインが数値と画像を出力するところまで」であり、差分率の低下は段階 5 (自動修正ループ) の仕事
