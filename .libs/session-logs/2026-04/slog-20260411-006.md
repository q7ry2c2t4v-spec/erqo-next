## SLOG-20260411-006.決定事項 — 決定事項

1. **Storybook 採用 + 最高精度を本気で追求** — VRT pixel-perfect モード (`threshold: 0, maxDiffPixels: 0`) で機械的検証
2. **DOM/CSS 抽出方式を採用** — 忠実度・保守性で優れる。Playwright CLI で実 DOM + 計算済み CSS を取得
3. **「本棚ページが正本」原則** — `.libs/storybook/<部品>/` 経由のみで修正、`.tsx` / `.stories.tsx` 直接編集禁止。理由: 二重情報源を避ける + 要望履歴の追跡可能性
4. **「元サイト < 要望」優先順位** — 取材した実値より、ユーザー要望が常に勝つ
5. **既存 /codi 5 ステップ維持、ステップ 2 だけ分岐** — 既存構造を壊さず通常タスクと共存。レイアウトタスクは 7 サブステップ (recon → dump → apply → rules → build → assemble → baseline) に展開
6. **新棚 `.libs/storybook/` 追加** — 1 部品 = 1 ページの正本。features 棚とは性質が違うため別棚
7. **Tailwind v4 `@theme` で全実値をトークン化** — 任意値 (`p-[17px]`) は禁止、元値を新規 token として登録 (例: `--spacing-extracted-17: 17px`)
8. **アニメも本棚正本に統合** — `element.getAnimations()` で取材、各 phase を独立 story 化、`prefers-reduced-motion` で VRT 用に停止
9. **B 4 項目 (shadcn/ui, framer motion, three.js, アニメ解析) の組み込み箇所を 7 サブステップに居場所確保** — 詳細は次回以降のリサーチで深掘り
10. **8 段階実装計画** — 準備 (済) → 判定機械化 → clone.py 骨格 → 7 サブステップ実装 → state.py 拡張 → SKILL.md 更新 → 動作確認 → giget 配布
