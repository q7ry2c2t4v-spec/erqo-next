## SLOG-20260412-003.概要 — 概要

/codi パイプラインのレイアウトタスク対応で 3 つの断絶を発見・修正した。(1) load.py がデザイン設計書を読み込まない、(2) record.py が設計書への参照を自動付与しない、(3) baseline が指示書を書くだけでスクショを撮らない。さらに VRT テスト基盤が根本的に未整備だったため、テストファイル雛形自動生成・Playwright VRT プロジェクト定義・テストディレクトリ自動作成まで一貫して接続した。Storybook を webpack ビルダーから Vite ビルダーに切り替え、Node 23 環境での動作も確保した。

## SLOG-20260412-003.作業内容 — 作業内容

### 1. load.py — 本棚ページの必須読み込み
- レイアウトタスク時に本棚ページ (.libs/storybook/<slug>/<slug>.md) の存在チェック + 全文読み込みを追加
- 本棚ページが存在しない場合はエラーで即 return（`/layo` を先に実行するよう案内）
- STORYBOOK_SHELF_NAME を constants.py から import

### 2. record.py — 設計書への参照自動付与
- _is_layout_task() を追加: タスク ID を LAYOUT_ID_PATTERNS と照合して機械判定
- _bookshelf_page_path() を追加: 本棚ページのパスを計算
- create_record() でレイアウトタスクかつ本棚ページが存在する場合、`関連:` に相対パスを自動付与

### 3. baseline.mjs — VRT 基準スクショ自動撮影
- core/clone_node/baseline.mjs を新規作成
- Storybook が未起動なら自動起動 → ストーリー iframe でスクショ → 停止
- Windows 対応 (shell: isWindows, taskkill /T /F)
- CI=true + STORYBOOK_DISABLE_TELEMETRY=1 でテレメトリプロンプト抑制
- reducedMotion: 'reduce' でアニメ停止

### 4. cmd_baseline 書き換え
- 指示書だけ → baseline.mjs を呼んで実際にスクショ撮影するように変更
- _find_baseline_script(), _run_node_baseline() を追加
- Story ID を PascalCase → Storybook ID 形式に自動変換

### 5. VRT テストファイル雛形の自動生成
- _build_vrt_spec_skeleton() を追加: tests/vrt/<slug>.spec.ts の雛形を生成
- _build_build_section() に VRT テスト雛形を追加（.tsx, .stories.tsx と並列）
- toHaveScreenshot() で pixel-perfect 比較

### 6. Playwright VRT プロジェクト定義
- specs/templates/playwright.config.ts に `vrt` (desktop 1440x900) + `vrt-mobile` (iPhone 14) プロジェクトを追加
- testDir: './tests/vrt' で VRT テスト専用ディレクトリを指定

### 7. install.py — テストディレクトリ自動作成
- STANDARD_DIRS に "e2e" と "tests/vrt" を追加

### 8. Storybook Vite ビルダー移行
- starter/package.json: @storybook/nextjs → @storybook/experimental-nextjs-vite
- starter/.storybook/main.ts: framework 名を変更
- starter/package.json: next を ~15.3.0 にピン (15.5.x で Storybook 互換性問題)
- Storybook バージョンを ^8.6.0 に統一

### 9. _build_baseline_section 整合
- 旧: 手動で npx playwright test を実行する手順書
- 新: baseline.mjs の自動撮影結果と整合した出力先・撮影条件の記録

### 10. skills/codi/SKILL.md コメント修正
- baseline サブステップのコメントを手動 → 自動実行の実態に修正

## SLOG-20260412-003.決定事項 — 決定事項

1. **load.py で本棚ページを必須読み込み** — 本棚ページが存在しないレイアウトタスクはエラーで停止。/layo の結果が /codi に確実に伝わる
2. **record.py で設計書参照を自動付与** — features ページの `関連:` に本棚ページの相対パスを機械的に追加。復元可能性の確保
3. **baseline は自動撮影** — 指示書方式を廃止。baseline.mjs が Storybook 起動 → スクショ → 停止を全自動で行う
4. **VRT テストファイルは cmd_build で雛形生成** — .tsx, .stories.tsx と並んで tests/vrt/<slug>.spec.ts も自動生成
5. **Storybook を Vite ビルダーに切り替え** — webpack ビルダーは Next.js 15.5+ / Node 23 で互換性問題あり。Vite ビルダーで 2 秒起動を実現
6. **Next.js は ~15.3.0 にピン** — 15.5.x で Storybook が参照する内部モジュール (define-env-plugin.js) が削除されたため
