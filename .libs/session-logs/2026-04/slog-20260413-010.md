## SLOG-20260413-010.決定事項 — 決定事項

### 1. 共有方式は「独立 repo + clone + 親 .gitignore」

submodule / subtree / subrepo / symlink / sync スクリプト など 6 候補を退けて 1 案を採用。
**理由:** 素人視点で「普通の git リポ」にしか見えず、復旧容易 + Windows/Mac 両対応 + 実装コスト最小。

### 2. 新棚 `.libs/research/os/` を新設 (webclone/ には入れない)

当初ユーザーは `webclone/` 配下を指示したが、「OS 自体の機能設計」はジャンル違いと判断し確認して `os/` に分離。
**理由:** 棚は意味的に分類するため、webclone (サイト再現) と os (メタ機能) を混ぜない。

### 3. マイグレーションは履歴破棄 (シンプル優先)

`git subtree split` による履歴保持を案として提示したが、履歴追跡の必要性が低いため破棄方式で新リポを `git init` + push。
**理由:** 研究ノートは参照が主で、コミット履歴より中身の方が重要。マイグレーション作業が 1 回だけなので短い手順を選ぶ。

### 4. `bootstrap.py` は変更しない

スタンドアロン制約があり独自定数を足す選択肢もあったが、`install.py` 経由で `setup_research_repo` が呼ばれるため不要と判断。
**理由:** DRY 原則。bootstrap が install を呼ぶ既存パターンにそのまま乗る。

### 5. `research_sync.py` に timeout を入れる

`session.py` が SessionStart Hook (timeout=10s) から呼ばれるため、`_GIT_TIMEOUT_SEC=10` を設定。clone のみ 60s。
**理由:** ネットワーク不通時にセッション起動を止めないため。タイムアウト時は疑似 CompletedProcess を返して degraded mode 続行。
