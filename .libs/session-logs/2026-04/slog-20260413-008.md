## SLOG-20260413-008.決定事項 — 決定事項

### 1. 段階 1 の合意事項を実装

- variant 値は `nextjs` / `generic` (案 2 採用、将来スタック追加時に拡張容易)
- スキル frontmatter に新フィールド `variant:` 追加 (target と直交、省略時 both)
- 印なしプロジェクトは nextjs 扱い (後方互換)
- 既存プル子は --update 時に自動マイグレーション (印なしなら nextjs 書き足し)

### 2. 段階 2 のスコープ境界

- Next.js 特化の **静的資産のみ** を stacks/nextjs/ に物理移動 (docs / starter / templates / skills/layo / l3-nextjs.md)
- `core/` の Python スクリプト (`clone.py` / `clone_node/` / `layo.py` / `docs.py`) は本元 core/ に残す
  - 理由: 本元 core モジュール群との import 関係を壊さないため。プタ子ではこれらは呼ばれないので実害なし
- `l3-cloudflare.md` は共通 (`specs/coding/`) に残す
  - 理由: Cloudflare Workers はプル子でもプタ子でも呼ぶ可能性あり、Next.js と一体ではない

### 3. リサーチ (`.libs/research/`) の三者共有は保留

- 現状は `docs/` のみ giget で共有されている (三者共有)、リサーチはプロジェクト毎独立
- 今セッションでは実装しない。必要性が出たら段階 5 以降で別案件で検討
