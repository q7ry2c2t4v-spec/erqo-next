## SLOG-20260413-006.決定事項 — 決定事項

1. **段階 4 は 2 リリース (C+A → WebGL 全 program) に分けた** — 前者は検証で bug 1 件 (rrweb × screenshot 干渉) を出したので早めに切って検証、後者は WebGL 専用で独立価値。段階投入方針を継続
2. **rrweb は scroll-samples 区間だけ起動** — goto 直後の起動は page.screenshot を timeout させる (MutationObserver + layout thrashing が fullPage screenshot と干渉)
3. **rrweb 起動 API を install/start/collect の 3 段階に分離** — UMD 注入と record 起動を分ける設計。外部から起動タイミング制御可能
4. **仮想時計 pause/resume 中は仮想時計を進めない** — scroll-samples 専用のエスケープ。pause→resume 間の実時計経過は仮想時計に加算しない
5. **WebGL hook は条件なしで install** — 段階 2 の CURRENT_PROGRAM フォールバックと両立するため、`__webglTrackedPrograms__` の件数で分岐させる
6. **shaders.json の programs[] に contextType を追加** — webgl/webgl2 の区別を明示。build 雛形は programs[0] を読むだけなので互換
7. **Vision LLM は段階 5 と組み合わせ** — 段階 4 の残り候補から除外。「AI が下書き → AI が差分で修正」を 1 セットで組む方が綺麗
8. **scroll-samples 純粋速度改善は先送り** — 段階 4-C で詰まりは解消済み、追加の純速度改善は実害なし
