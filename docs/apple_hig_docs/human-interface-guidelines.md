# ヒューマンインターフェイスガイドライン

「ヒューマンインターフェイスガイドライン」には、どのAppleプラットフォームでも優れた体験を設計できるようにするためのガイドとベストプラクティスが含まれています。

Appleプラットフォーム用のインターフェイスをデザインする際は、次の原則を十分に考慮してください:

![カプセル型の縦に並んだ項目グループの先頭側に2つの円形ボタンが縦に並んでいる概念図。これら3つの要素は、より大きな角丸正方形の上に配置されています。](https://docs-assets.developer.apple.com/published/214efefd347927407f3273dedbd8df7f/homepage-establish-hierarchy%402x.png)

#### [階層](/jp/design/human-interface-guidelines#Hierarchy)

コントロールとインターフェイス要素が、その下にあるコンテンツを強調し、区別する、明確な視覚的階層を確立します。

![カプセル型の項目グループの先頭側に配置された円形ボタンの概念図。](https://docs-assets.developer.apple.com/published/eda7850fbadba354dbdf4be2f926d337/homepage-create-harmony%402x.png)

#### [調和](/jp/design/human-interface-guidelines#Harmony)

ハードウェアとソフトウェアの同心円状のデザインに合わせて、インターフェイス要素、システム体験、およびデバイス間の調和を生み出します。

![円形ボタンの概念図。](https://docs-assets.developer.apple.com/published/0f54ed63439a729e27d8d605d2b6a5b4/homepage-maintain-consistency%402x.png)

#### [一貫性](/jp/design/human-interface-guidelines#Consistency)

プラットフォームで使用されている規則を採用して、ウインドウサイズやディスプレイに継続的に適応する一貫性のあるデザインを維持します。

デベロッパ向けのガイダンスは、[Liquid Glassの採用](/documentation/TechnologyOverviews/adopting-liquid-glass)を参照してください。
