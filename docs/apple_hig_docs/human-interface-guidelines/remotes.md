# リモコン

Siri RemoteはApple TVのプライマリとなる入力手段です。これを使用することで、ユーザは部屋のどこからでも画面のコンテンツを手軽に操作することができます。

![Apple TV Remoteのスケッチ。画面のコンテンツの操作を示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせる紫色になっています](https://docs-assets.developer.apple.com/published/07f1af0485ecc7a490b62aceae28fc4d/inputs-remotes-intro%402x.png)

Siri Remoteは、いくつかの固有のボタンと、クリックパッドおよびタッチサーフェスを組み合わせたものです。クリックパッドとタッチサーフェスを使用すると、押したりスワイプしたりといったおなじみのジェスチャで、tvOSアプリの操作、チャンネルやコンテンツのブラウズ、メディアの再生や一時停止、選択などを行うことができます。

## [ベストプラクティス](/jp/design/human-interface-guidelines/remotes#Best-practices)

**標準のアクションには標準のジェスチャを使用する。** ゲームのプレイ中を除けば、ユーザは使用するすべてのアプリでRemoteが標準的な方法で動作することを期待します。標準のRemoteの動作を再定義したり、別の目的に使用したりすると、操作が複雑になり、ユーザが戸惑う場合があります。ガイダンスは[ジェスチャ](/jp/design/human-interface-guidelines/remotes#Gestures)を参照してください。

**tvOSのフォーカス操作と一貫性を持たせる。**[フォーカス操作](/jp/design/human-interface-guidelines/focus-and-selection)は、ユーザと視聴中のコンテンツとの間に強力な一体感を生み出します。アプリでこの一体感を強化できるように、ユーザが慣れ親しんだ方法でジェスチャとフォーカス操作を組み合わせましょう。例えば、必ずジェスチャと同じ方向にフォーカスが移動するようにしてください。

**アプリでジェスチャを行うと何が起こるのかを示す明確なフィードバックを提供する。** 例えば、ユーザがRemoteに軽く親指を乗せたときに、どこを下にスワイプすれば情報領域が表示されるかを示します。

**特に理由がない限り、アプリで新しいジェスチャを定義しない。** 例えばゲームであれば、カスタムのジェスチャが楽しい体験を生み出すかもしれません。しかし、それ以外のほとんどの場合、ユーザは標準のジェスチャを使用できることを期待し、新しいジェスチャを探したり覚えたりしたいとは思いません。

**押す操作とタップ操作は区別し、意図しないタップに反応しない。** 押す操作は意図的なアクションであるため、ボタンの選択、選択の確定、ゲーム中のアクションの開始などの操作に適しています。タップジェスチャはナビゲーションや追加情報の表示には向いていますが、Remoteに親指を乗せる、Remoteを持ち上げる、移動させる、誰かに渡すなどの際に意図せずにタップされる場合があることに注意してください。ほとんどの場合、ビデオの再生中はタップに反応しない方がよいでしょう。

**タップ位置をナビゲーションやゲームに使用することを検討する。** Remoteのタッチサーフェスのタップジェスチャでは、上下左右が区別されます。この操作が、アプリの内容に適していて、直感的で分かりやすい方法で実現可能である場合に限り、位置を示すタップを利用するようにしてください。

**ほとんどの場合、「戻る」ボタンが押されたときは現在の画面の親画面を開く。** アプリやゲームのトップレベルでは、親はApple TVのホーム画面です。アプリの中では、親はアプリの階層によって決まりますが、必ずしも前の画面とは限りません。この標準動作の例外は、ゲームのプレイ中です。ゲーム中は、意図せずに何度も「戻る」ボタンが押される可能性があります。この場合は、ゲームプレイの品質を損なわないように、「戻る」ボタンが押されたらゲーム内一時停止メニューを開き、別の操作を行ってゲームのメインメニューに戻れるようにします。ゲーム内一時停止メニューが開いているときに「戻る」ボタンが押されたら、メニューを閉じてゲームを再開します。「戻る」ボタンを長押しすると、どの場所からでもホーム画面に戻れる点に注意してください。ガイダンスは、[ボタン](/jp/design/human-interface-guidelines/remotes#Buttons)を参照してください。

**メディア再生中の再生/一時停止ボタンに適切に反応する。** ユーザは、ミュージックやビデオの再生中に、再生/一時停止ボタンを押して再生、一時停止、再開できることを期待します。

## [ジェスチャ](/jp/design/human-interface-guidelines/remotes#Gestures)

クリックパッドのタッチサーフェスは、スワイプと押す操作を検知します。

**スワイプ:** スワイプによって、多数の項目を簡単にスクロールできます。スワイプの強さに応じて、最初は速く、その後ゆっくりとスクロールします。Remoteの端を上下にスワイプすると、非常に高速に項目を移動できます。

**押す:** 押す操作によって、コントロールを作動させたり、項目を選択したりします。また、押す操作のあとにスワイプを行うと、スクラブモードが有効になります。

## [ボタン](/jp/design/human-interface-guidelines/remotes#Buttons)

アプリやゲームで特定のボタンが押された場合、次のように対応してください。

ボタン/領域| アプリで期待される動作| ゲームで期待される動作  
---|---|---  
タッチサーフェス（スワイプ）| 項目間を移動する。フォーカスを変更する。| 方向パッドの動作を実行する。  
タッチサーフェス（押す）| コントロール/項目を作動させる。さらに詳細な項目に移動する。| 主ボタンの動作を実行する。  
戻る| 前の画面に戻る。Apple TVのホーム画面に戻る。| ゲームプレイを一時停止/再開する。前の画面/ゲームのメインメニュー/Apple TVのホーム画面に移動する。  
再生/一時停止| メディアの再生を開始する。メディアの再生を一時停止/再開する。| 副ボタンの動作を実行する。導入部分のビデオをスキップする。  
  
## [互換リモコン](/jp/design/human-interface-guidelines/remotes#Compatible-remotes)

Apple TVと互換性のあるリモコンには、ライブTVやその他のチャンネルのコンテンツをブラウズするボタンが含まれていることがあります。例えば、リモコンに電子番組ガイド（EPG）を開くボタンや、ガイドのブラウズまたはチャンネルの変更に使用するボタンが含まれている可能性があります。デベロッパ向けのガイダンスは[チャンネルナビゲーションの提供](/documentation/TVServices/providing-channel-navigation)、デザインのガイダンスは[EPG操作](/jp/design/human-interface-guidelines/live-viewing-apps#EPG-experience)を参照してください。

**ライブビューイングアプリでEPGを提供する場合は、ユーザが期待する形でリモコンのEPGブラウズボタンに反応させる。** 「ガイド」または「ブラウズ」ボタンを押すとき、ユーザはEPGが開くことを期待します。EPGを表示している間は、「ページアップ」または「ページダウン」ボタンを押して移動できることを期待します。ユーザがEPGをブラウズしている間は、これらのボタンに別の方法で反応させないようにしてください。Siri Remoteと互換リモコンでは、タッチサーフェスの上部または下部をタップしてEPGをブラウズできることが期待されます。EPGに対応しないアプリでこれらのボタンが押されると、視聴者のデバイスのデフォルトのガイドアプリが反応します。

**コンテンツの再生中に互換リモコンの「ページアップ」または「ページダウン」ボタンが押された場合は、チャンネルを変更する。** ユーザは、コンテンツの視聴中とEPGのブラウズ中で、これらのボタンの動作が異なることを期待します。
