> Source: https://developer.apple.com/jp/design/human-interface-guidelines/activity-views

# アクティビティビュー

アクティビティビューは _共有シート_ とも呼ばれ、現在のコンテキストで実行できるさまざまなタスクが表示されます。

![図案化されたアクティビティビューまたは共有シート。画像は全体が、6色のオリジナルAppleロゴを連想させる赤色になっています。](https://docs-assets.developer.apple.com/published/ff4abe706ca9e38c5ff145f682055a03/components-activity-view-intro%402x.png)

アクティビティビューから、よく使うアプリに素早くアクセスできるほか、メッセージなどの共有アクティビティ、コピーやプリントなどのアクションを実行できます。通常は、ページや書類を表示しているときや項目を選択したあとに、アクションボタンを押して共有シートを表示します。デバイスの種類や向きによって、アクティビティビューはシートとして表示されることも、ポップオーバーとして表示されることもあります。

ユーザがアプリやゲームで開く共有シートに、アプリ固有のアクティビティを表示させることもできます。例えば「写真」では、「写真をコピー」、「アルバムに追加」、「位置情報を調整」などのアプリ固有のアクションが表示されます。アプリ固有のアクションは、デフォルトでは、複数のアプリやシステム全体で利用できるアクション（「ファイル」への追加やAirPlayなど）よりも前に表示されます。ユーザはアクションのリストを編集して、よく使う項目を優先して表示したり、新しい項目を追加したりできます。

アプリ機能拡張を作成して、ほかのアプリで使用できるカスタム共有やアクションアクティビティを提供することもできます。（ _アプリ機能拡張_ は、ユーザがインストールしてアプリの外部で使用できるコードを指します。）例えば、Webページを特定のソーシャルメディアサービスで簡単に共有できるカスタム共有アクティビティを作成し、ユーザにインストールを促す、という設計も考えられます。アクティビティビューはmacOSには対応していませんが、Macで使用できる共有およびアクションのアプリ機能拡張は作成できます。ガイダンスは[共有機能拡張とアクション機能拡張](/jp/design/human-interface-guidelines/activity-views#Share-and-action-extensions)を参照してください。

## [ベストプラクティス](/jp/design/human-interface-guidelines/activity-views#Best-practices)

**アクティビティビューですでに利用できる一般的なアクションと重複するアクションを作らない。** 例えば、「プリント」アクションと重複するアクションは不要な上に、システムが提供するアクションと見分けがつかず、ユーザを戸惑わせることになります。既存のアクションに似たアプリ固有の機能を提供する場合は、カスタムのタイトルを指定してください。例えば、銀行の取引明細をプリントする際にカスタムの書式設定を使用できるようにする場合は、「取引明細のプリント」のように、アプリのプリントアクティビティの処理内容を理解できるようなタイトルにします。

**カスタムのアクティビティをシンボルで表現することを検討する。**[SF Symbols](/jp/design/human-interface-guidelines/sf-symbols)にある設定可能なさまざまなシンボルを使って、アクティビティビューの項目や概念を表せます。カスタムのインターフェイスアイコンを作成する必要がある場合は、およそ70x70ピクセルの領域の中心に配置します。ガイダンスは[アイコン](/jp/design/human-interface-guidelines/icons)を参照してください。

**カスタムアクションごとに簡潔で分かりやすいタイトルを付ける。** タイトルが長すぎると、折り返されたり、切れたりする場合があります。できる限り、アクションの内容を明確に伝える単一の動詞や簡潔な動詞句を使用してください。アクションのタイトルには会社名や製品名を含めないでください。一方、共有シートで共有アクティビティのアイコンの下に表示されるタイトルは、基本的に会社名です。

**現在のコンテキストに適したアクティビティであることを確認する。** アクティビティビューでシステムが提供するタスクを並べ替えることはできませんが、アプリに適さないタスクは除外できます。例えば、アプリからプリントする必要がない場合は、「プリント」アクティビティを除外できます。どのタイミングでどのカスタムタスクを表示するかを指定することもできます。

**アクティビティビューは「共有」ボタンからのみ表示する。** システムが提供するアクティビティを利用するユーザは、通常、「共有」ボタンを押します。同じ動作を別の操作で実行できるようにすると、ユーザが戸惑うだけです。

## [共有機能拡張とアクション機能拡張](/jp/design/human-interface-guidelines/activity-views#Share-and-action-extensions)

共有機能拡張を使用すると、ユーザは現在のコンテキストの情報を、別のアプリ、ソーシャルメディアのアカウント、その他のサービスに簡単に共有できます。アクション機能拡張を使用すると、ユーザは現在のコンテキストを離れることなく、ブックマークを追加する、リンクをコピーする、インライン画像を編集する、選択したテキストを別の言語で表示するなど、コンテンツ固有なタスクを開始できます。

共有機能拡張とアクション機能拡張は、プラットフォームによって異なる形で表示されます。

  * iOSとiPadOSでは、アクションボタンを押したときに表示される共有シートに共有機能拡張とアクション機能拡張が表示されます。

  * macOSでは、ツールバーの「共有」ボタンをクリックまたはコンテキストメニューの「共有」を選択したときに、共有機能拡張にアクセスできます。アクション機能拡張には、特定のタイプの埋め込みコンテンツ（「メール」の作成ウインドウに追加した画像など）にポインタを合わせるか、ツールバーのボタンをクリックするか、Finderウインドウでクイックアクションを選択してアクセスします。

**必要に応じて、ユーザが使いやすいと感じるカスタムインターフェイスを作成する。** 共有機能拡張では、なるべくシステムが提供するビューを使用して、ユーザに馴染みのある一貫した共有操作環境にします。アクション機能拡張には、アプリ名を含めます。インターフェイスを表示する必要がある場合は、ユーザが機能拡張とアプリの関連性を理解できるように、アプリのインターフェイス要素を含めるようにします。

**操作を合理化して手順を減らす。** ユーザは、数ステップでタスクを完了できる機能拡張を高く評価します。例えば、共有機能拡張では、1回のタップやクリックですぐにソーシャルメディアアカウントに画像を投稿できるようにします。

**機能拡張にモーダルビューを重ねて配置しない。** デフォルトでは、機能拡張はモーダルビューに表示されます。機能拡張にアラートを重ねて表示しなければならない場合はありますが、追加でモーダルビューを表示することは避けてください。

**必要に応じて、機能拡張の目的を伝える画像を用意する。** 共有機能拡張では、機能拡張を提供しているアプリがユーザに明確に分かるように、アプリアイコンが自動的に使用されます。アクション機能拡張では、[シンボル](https://developer.apple.com/jp/design/human-interface-guidelines/sf-symbols)の使用、またはタスクを明確に識別できるインターフェイス[アイコン](https://developer.apple.com/jp/design/human-interface-guidelines/icons)の作成を優先して検討してください。

**時間がかかる操作の進捗はメインアプリで伝える。** 共有機能拡張やアクション機能拡張では、ユーザがタスクを終えるとすぐにアクティビティビューが閉じます。タスクに時間がかかり、バックグラウンドでタスクが継続される場合は、メインアプリで状況を確認する方法を提供します。通知を使用して問題を知らせることはできますが、タスクが完了したことを伝えるためだけに通知は使わないようにしてください。
