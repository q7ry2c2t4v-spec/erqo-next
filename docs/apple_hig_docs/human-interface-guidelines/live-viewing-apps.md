# ライブビューイングアプリ

ライブビューイングアプリを設計するときは、ライブビューイング体験のイマーシブ感を高めるために、コンテンツと快適でスムーズな操作性を優先します。

![テレビに再生ボタンが合わさっているスケッチ。メディアの再生を示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/5293a6e0ba4a98fa2e6a10568b4bdbea/patterns-live-viewing-intro%402x.png)

ライブビューイングアプリは、ライブコンテンツであることを最優先に考慮する必要があります。どの画面でもユーザをライブコンテンツに惹きつけ、ビデオオンデマンド（VOD）コンテンツとの違いが一目で分かるようにしなければなりません。

## [ベストプラクティス](/jp/design/human-interface-guidelines/live-viewing-apps#Best-practices)

**ライブコンテンツが目立つように表示し、すぐにアクセスできるようにする。** ユーザはコンテンツを観るためにアプリを使用します。アプリの起動からコンテンツ再生までの時間をできるだけ短くすることをおすすめします。最初のタブにライブコンテンツがあれば、コンテンツの視聴を開始するために何度もタップする必要がありません。

**1回のタップ操作またはタップ操作なしで再生を開始できるようにする。** 例えば、おすすめのライブコンテンツや最近視聴したライブコンテンツの上に、「今すぐ観る」ボタンを表示しておくのはよい方法です。ユーザがこのボタンをタップすると、ボタンがすぐに消えて再生が始まります。このタイミングでアプリのUIを消し、イマーシブ感の高いフルスクリーン視聴体験を提供します。

**ライブコンテンツであることがはっきりと分かるようにする。** ユーザがライブコンテンツとVODコンテンツを見分けられるようにする必要があります。ライブコンテンツを単純に再生するのがライブ感を出すシンプルな方法とはいえ、ライブコンテンツを見分けやすくするためのマーキング方法はいくつかあります。例えば、「ライブ」というタイトルのコレクション行にほかのチャンネルをまとめて表示し、それぞれの項目にバッジ、シンボル、帯などを付けて、ライブであることを視覚的に示すことができます。

**現在再生中のライブコンテンツの進行状況を示す。** 進行中のライブコンテンツを途中から視聴し始めるユーザは、今コンテンツのどのあたりであるか知りたいはずです。進行状況バーなどのインジケータを使って、コンテンツの残り時間をユーザに通知できます。

**ユーザに追加の操作と代替の視聴手段を提供する。** 主要な操作である再生はもちろんですが、それ以外の録画、再開、ダウンロードなど、アプリで対応するアクションを簡単に実行できるようにしてください。また、これらの操作は例えば、「視聴」、「最初からやり直す」、「録画」、「お気に入り」のように、アプリ全体を通して同じ順序で表示します。さらに、現在再生中のコンテンツをほかの時間にもう一度再生する場合は、ユーザが視聴をスケジュールできるようにこの情報を表示します。

**再生中にチャンネルのブラウズができるようにコンテンツフッタの利用を検討する。** コンテンツフッタを使えば、ユーザはライブ再生中のコンテンツを視聴したままチャンネルをブラウズできます。コンテンツフッタを使用する場合は、以下の点に注意してください:

  * テキストの可読性を維持するために文字の背景を暗くするなどさりげない調整を行い、背後で再生中のコンテンツからすべての項目を容易に区別できるようにします。

  * サムネールにバッジを付ける、進行状況バーの色合いを調整するなど、現在再生中のコンテンツを表すサムネールをユーザが簡単に識別できるようにします。

  * コンテンツフッタのカテゴリを電子番組ガイドのカテゴリに合わせます（関連するガイダンスは、[EPG（電子番組ガイド）](/jp/design/human-interface-guidelines/live-viewing-apps#EPG-experience)を参照してください）。

  * ユーザがコンテンツフッタを呼び出したり消したりするためのシンプルで推測しやすい方法を提供します。例えば、上にスワイプするとフッタが呼び出される場合、ユーザは、下にスワイプすればフッタを消せると考えます。

**ユーザがチャンネルを変えたら即座に視覚的フィードバックを返す。** これは2つの理由で重要です。まず、ユーザは自分が見たいチャンネルに到達したことを把握できる必要があります。また、フィードバックを提供している間に、ストリーミングコンテンツを読み込むことができます。

**現在の視聴状況に合ったオーディオを再生する。** ライブコンテンツの再生を開始したユーザは、ブラウズに切り替えてコンテンツがバックグラウンド再生になっても、ライブコンテンツのオーディオが再生されることを期待します。これに対し、ユーザがアプリのライブタブから移動したらライブコンテンツの視聴も終了するので、オーディオも停止する必要があります。

## [EPG（電子番組ガイド）](/jp/design/human-interface-guidelines/live-viewing-apps#EPG-experience)

一般に、ライブビューイングアプリでは、スケジュールされた番組の情報が分かる電子番組ガイド（EPG）が提供されます。ライブビューイングアプリの特性に合わせた、ユーザにとって使いやすいEPGを提供するため、以下のガイドラインに従ってください。

**現在の情報を目立つように表示して再生に簡単に戻れるようにする。** ユーザが初めてEPGを開いたときに元のチャンネルにすぐに戻れるように、現在の番組、チャンネル、時間を簡単に見つけられるようにする必要があります。

**EPGを簡単にブラウズできるようにする。** 一般にEPGには大量の情報が含まれているので、ユーザがページ表示、スクロール、ジャンプを簡単に行えるようにするのは大切です。「マイチャンネル」グループやお気に入りのグループを提供して、よく視聴するコンテンツにすぐにアクセスできるようにするのも一案です。

**コンテンツをユーザにとって分かりやすいカテゴリに分類して見つけやすくする。** 例えば、映画、テレビ番組、子供向け、スポーツ、人気などのカテゴリが考えられます。アプリでコンテンツフッタに対応している場合、EPGと同じカテゴリを使ってコンテンツのサムネールを編成してください。

**ユーザが現在のコンテンツを視聴したままEPGをブラウズできるようにする。** 例えば、ユーザがEPGをブラウズしている間、ピクチャインピクチャ（PiP）モードやバックグラウンドでコンテンツの再生を継続できます。

## [クラウドDVR](/jp/design/human-interface-guidelines/live-viewing-apps#Cloud-DVR)

アプリがクラウドでのデジタルビデオ録画（DVR）に対応する場合は、以下のガイドラインに従ってください。ライブビューイングアプリで優れた録画機能を実現できます。

**情報パネルから録画の開始/停止を操作できるようにする。** ライブストリーミング中にすぐに録画を開始できる情報パネルを表示できると、ユーザの利便性が向上します。

**コンテンツの詳細情報を表示するビューで将来の番組を録画できるようにする。** その番組のみの録画か、将来のすべてのエピソードの録画かを選択できるオプションも提供してください。

**ユーザのニーズに合わせて録画方法を調整できるようにする。** 例えば、現在のエピソードのみ、新しいエピソードのみ、特定のチームが出ている試合のみなど、録画したい内容をユーザが正確に指定できるようにします。

**クラウドDVRセクションで再生およびコンテンツ固有のほかのアクションを行えるようにする。** ユーザがクラウドDVRセクションでコンテンツの詳細情報を表示するビューを開いたときに、コンテンツの再生や削除に加えて、適切な場合には録画設定を調整できるようにもします。

**ユーザがクラウドDVR設定を管理できるコントロールを提供する。** 例えば、視聴済みの録画や指定した日数が経過したコンテンツを削除できるようにします。最も古いコンテンツや視聴済みのコンテンツを上書きする自動ストレージ管理をユーザが設定できるようにして、領域不足を回避できるように支援できれば理想的です。
