# アクションボタン

アクションボタンによって、対応しているiPhoneやApple Watchのモデルで、よく使う機能をすぐに実行することができます。

![Apple Watchのアクションボタンに向いている矢印のスケッチ。アクションの開始を示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせる紫色になっています](https://docs-assets.developer.apple.com/published/f2a26ecd77388f99d4a84188379ca385/inputs-action-button-intro%402x.png)

対応しているデバイスでは、アクションボタンを使用して、[アプリショートカット](/jp/design/human-interface-guidelines/app-shortcuts)を実行したり、システムで提供される機能（フラッシュライトのオン/オフ切り替えなど）にアクセスしたりできます。Apple Watch Ultraのアクションボタンは、ワークアウトやダイビングなどのアクティビティ関連のアクションに対応しています。

ユーザは、デバイスを設定する際にアクションボタンの機能を選択でき、その後は「設定」でその選択を調整できます。アクションボタンにアプリショートカットを関連付けると、Siriで音声を使用したり、Spotlightでタップしたりするのと同様に、ボタンを押すことでアプリショートカットが実行されます。

アプリやゲームを設計するときに、アクションボタンは、よく使う機能を素早く実行するためのもう1つの方法として考えてください。

## [ベストプラクティス](/jp/design/human-interface-guidelines/action-button#Best-practices)

**アクションボタンはアプリの本質的な機能に対応させる。** 例えば、料理アプリにゆで卵用タイマーが含まれている場合、ユーザがアクションボタンを押したときに開始したいアクションは「ゆで卵用タイマーを開始する」である可能性があります。アプリを開くアプリショートカットは、すでにシステムで提供されているので、提供する必要はありません。アプリのアイコン、ウィジェット、およびApple Watchのコンプリケーションによって、アプリを素早く開くための別の方法がユーザに提供されます。ガイダンスは、[アプリショートカット](/jp/design/human-interface-guidelines/app-shortcuts)も参照してください。

**対応しているアクションを簡潔に説明する短いラベルを使用する。** ユーザは「設定」にアクセスしたときに、ラベルを見て、アクションボタンの動作を設定します。[タイトルにふさわしいスタイル](https://support.apple.com/guide/applestyleguide/c-apsgb744e4a3/web#apdca93e113f1d64)を使用し、動詞で始まり、現在形を使用し、冠詞や前置詞を含まないラベルを作成します。ラベルは可能な限り短くし、3語以内にします。例えば、「Started Race」や「Start the Race」ではなく「Start Race」を使用します。

**アプリで対応したアクションボタンの使用方法をシステムに表示させる。** アプリがアクションボタンに対応している場合は、アクションボタン経由でアプリの機能を開始するための設定に関するヘルプが自動的に表示されます。アクションボタンの「設定」に表示されるガイダンスや、システムに用意されているその他の使い方のヒントを繰り返すようなコンテンツは作成しないでください。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/action-button#Platform-considerations)

 _iPadOS、macOS、tvOS、visionOSには対応していません。_

### [iOS](/jp/design/human-interface-guidelines/action-button#iOS)

**ユーザが現在のコンテンツから離れずにアクションを使用できるようにする。** 可能な場合は、[ライブアクティビティ](/jp/design/human-interface-guidelines/live-activities)やカスタムスニペットなどの軽量なマルチタスク機能を利用して、アプリを開かずに機能を実行できるようにします。例えば、「タイマーを設定」アクションでは、時計アプリを起動しません。ユーザにタイマーの時間を設定するように求めたあと、カウントダウンと共にライブアクティビティを起動します。

### [watchOS](/jp/design/human-interface-guidelines/action-button#watchOS)

watchOSでは、最初にアクションボタンが押されたときのアクションをユーザが割り当てることができます。これにより、ウェイポイントをドロップしたり、ダイビングを開始したり、特定のワークアウトを開始したりできます。アクションボタンでは、1回アクションボタンが押されたあと、セカンダリアクションに対応することもできます。例えば、複数のパートに分かれたワークアウトの実施中にセグメントをマークしたり、次のモダリティに移行したりできます。

**ユーザが選択したプライマリアクションをあと押しするようなセカンダリ機能の提供を検討する。** ユーザは画面を見ずにアクションボタンを使用することが多いため、2回目以降にボタンを押す操作では、最初のボタン操作との論理的整合性を持ちながら、現在のコンテキストに適合する機能を提供する必要があります。アプリがワークアウトやダイビングといったアクションに対応している場合、ユーザが簡単に使い道を学んで覚えることができる、シンプルで直感的なセカンダリ機能を設計することを検討してください。セカンダリ機能が複数あると、ユーザの認知負荷が増え、アプリが使いづらいと思われるおそれがあるため、事前に十分に検討してから提供するようにしてください。

**2回目以降にボタンが押されたときは、機能を停止または終了するのではなく、追加機能に対応するようにする。** ユーザが（現在の機能を停止するのではなく）主要なタスクを停止できるようにする必要がある場合は、アプリのインターフェイスを通じてそのオプションを提供するようにしてください。

**ユーザがアクションボタンとサイドボタンを同時に押したときは現在の機能を停止する。** ダイビング用のアプリは例外です。ダイビング機能を停止すると、現在の水深や水中での経過時間が分からなくなるため、ダイバーにとって危険な状況になる場合があります。現在の機能を停止してもネガティブな体験につながらない場合は、必ずユーザの期待に応じ、2つのボタンを同時に押したときに現在のアクティビティを停止できるようにしてください。
