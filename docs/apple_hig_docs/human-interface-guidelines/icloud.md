# iCloud

iCloudは、どのデバイスからでもシームレスに写真、ビデオ、書類などの重要なコンテンツにアクセスできるサービスです。明示的に同期を行う必要はありません。

![iCloudアイコンのスケッチ。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせる青色になっています。](https://docs-assets.developer.apple.com/published/f096c4727e340be72492543363a9bc15/technologies-iCloud-intro%402x.png)

iCloudの基本的な特徴は処理が自動化されていることです。ユーザはコンテンツがどこにあるかを知る必要はありません。常に最新版にアクセスできる、とだけ考えておけばよいのです。

## [ベストプラクティス](/jp/design/human-interface-guidelines/icloud#Best-practices)

**アプリをiCloudと連係しやすくする。** ユーザは、「設定」でiCloudをオンにしていれば、アプリも自動的にiCloudと連係することを期待します。ユーザがアプリでiCloudを使うかどうかを選択できるようにしたい場合は、初めてアプリを開いたときに簡単なオプションを表示して、すべてのデータでiCloudを使用するか、まったく使用しないかを選択できるようにしてください。

**iCloudにどの書類を保存するかを尋ねない。** ほとんどのユーザは、すべてのコンテンツがiCloudで利用できることを期待し、個々の書類の保存状態を管理したいとは思いません。アプリでどのようにコンテンツを扱い、共有するかを検討し、できるだけファイル管理タスクを自動化するようにしましょう。

**可能な限りコンテンツを最新に保つ。** iCloud対応アプリでは、常に最新のコンテンツにアクセスできるのが理想です。ただし、これに関してはデバイスのストレージや帯域幅の制約を考慮に入れる必要があります。アプリで非常に大量の書類を扱う場合は、アップデートされたコンテンツをダウンロードするタイミングを制御できるようにする方がよいでしょう。この種のアプリでは、iCloudに最新版の書類があることを示す方法を検討してください。書類がアップデート中で、ダウンロードに数秒以上かかる場合は、何らかのフィードバックを控えめに表示します。

**iCloudストレージの容量を考慮する。** iCloudは、有料かつ有限のリソースです。iCloudは、ユーザ自身が作成した情報や把握している情報を保存するために使用し、アプリのリソースや再生成可能なコンテンツは保存しないようにします。iCloudに対応しないアプリでも、iCloudバックアップにはすべてのアプリの「書類」フォルダに保存されたコンテンツが含まれる点に注意してください。容量を使いすぎないようにするために、「書類」フォルダに格納するコンテンツはよく選びましょう。

**iCloudが利用できないときでもアプリが適切に動作するようにする。** 手動でiCloudをオフにしたり、機内モードをオンにしたりした場合は、iCloudが利用できないことを通知する必要はありません。ただし、iCloudにアクセスできるようになるまで、行った変更はほかのデバイスに反映されないことを邪魔にならない形で知らせるとよいかもしれません。

**アプリのステータス情報を継続的にiCloudに保存する。** iCloudには、書類などのファイルだけでなく、設定やアプリの状態に関する情報を保存することもできます。例えば雑誌アプリなら、表示した最後のページを保存しておくと、ほかのデバイスでアプリが開かれたときに、前回閉じた場所から続きを読むことができます。iCloudに設定を保存する場合は、すべてのデバイスに適用したいとユーザが考える設定であることを確認してください。例えば、自宅よりも職場での方が便利な設定もあるかもしれません。

**書類の削除結果について警告する。** iCloud対応アプリで書類を削除すると、iCloudからだけでなく、すべてのデバイスから削除されます。削除を行う前に、確認の警告を表示してください。

**簡単かつ即座に競合を解消できるようにする。** 可能な限り、バージョンの競合を自動的に検出して解消するようにします。これができない場合は、邪魔にならない通知を表示して、競合するバージョンを簡単に区別して選択できるようにします。間違ったバージョンに無駄な時間を費やすことがないように、競合の解消は可能な限り早い段階で行うのが理想的です。

**検索結果にiCloudのコンテンツを含める。** iCloudアカウントを持つユーザは、iCloudのコンテンツはどのデバイスからでも利用可能と考えます。検索結果にも同様のことが期待されます。

**ゲームではプレイヤーの進行状況をiCloudに保存することを検討する。** この機能は独自に実装することもできますが、GameSaveフレームワークという効率的なソリューションも提供されています。このフレームワークはセーブデータをデバイス間で同期し、オフラインプレイ中やデータ競合が発生したときの同期の問題にプレイヤーが対処できるように、アラート機能も備えています。または、GameSaveデータを使用するカスタムUIを実装して、こうした状況に対応することもできます。デベロッパ向けのガイダンスは、[GameSave](/documentation/GameSave)を参照してください。
