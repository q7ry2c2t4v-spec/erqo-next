# 取り消し/やり直し

「取り消し/やり直し」は、さまざまな操作を簡単に元に戻せる方法です。ユーザが新しいインターフェイスやタスクを学習する際にさまざまな操作を安全に試すうえでも役立ちます。

![右向きで始まり、上に向かってカーブし、左を指す矢印のスケッチ。最初からのやり直しを示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/fa3bc73f68b584a62b26694855ae1caa/patterns-undo-redo-intro%402x.png)

ユーザは、取り消し/やり直しを使って直前の操作を元に戻せることを期待して、目的の変更内容にたどり着くまで（通常は何度も）取り消そうとします。このような状況で、ユーザは取り消しの対象となるのが以前のどの操作か明確でない場合があり、結果意図しない変更につながったり、ストレスを感じてしまう可能性があります。ユーザが状況を常に把握できるよう、取り消し/やり直しの操作の結果を予測しやすくすることと、操作の結果を強調表示することがきわめて大切です。

## [ベストプラクティス](/jp/design/human-interface-guidelines/undo-and-redo#Best-practices)

**可能な限りユーザが取り消し/やり直し操作の結果を予測できるようにする。** 例えばiPhoneでは、ユーザがデバイスをシェイクしたときに表示される通知で結果を示して、取り消しを実行するかキャンセルするかを選択できるようにします。取り消し/やり直しのメニュー項目を表示する場合は、どのような結果になるのかわかるようにメニュー項目ラベルを調整できます。例えば書類ベースのアプリでは、「取り消す - テキスト入力」や「やり直す - ボールド」のようなメニュー項目ラベルを使用できます。

**取り消し/やり直しの結果を示す。** ユーザが取り消したいと思っている直前の操作が、現在は表示されていないコンテンツや領域を対象にしている場合があります。このような場合は、ユーザが取り消し/やり直し操作をしても反映されていないと思って取り消し/やり直し操作を何度も実行しないように、個々の取り消し/やり直しの結果を強調表示することがきわめて重要です。例えば、ユーザが現時点で画面に表示されていない書類の領域で段落を削除したあとに取り消しを実行した場合は、書類をスクロールして復元された段落を表示するようにします。

**ユーザが何度も取り消しできるようにする。** ユーザが取り消し/やり直しできる回数に不要な制限を設けないでください。ユーザは一般に、書類を開く、作業を保存するといった論理的な手順を実行したあとのすべての操作を取り消せることを期待します。

**複数の変更をまとめて元に戻すオプションの提供を検討する。** シナリオによっては、関連する個々の操作をまとめて取り消す機能がユーザに喜ばれることもあります。例えば、同じプロパティや属性に対して徐々に変更を加えた場合などであれば、個々の変更を取り消す手間が省けます。または、書類を開いたあとや作業を保存したあとに行ったすべての変更をまとめて取り消す便利な方法を提供するとよい場合もあります。

**取り消し/やり直しボタンは必要な場合にのみ提供する。** ユーザは一般に、macOSアプリの「編集」メニューで各項目を選択する、MacまたはiPadでキーボードショートカットを使用する、iPhoneをシェイクするなど、システムが対応している方法で取り消し/やり直しを開始します。アプリ内で専用の取り消し/やり直しボタンを追加する場合は、システムで提供されている標準の記号を使用し、ツールバーにボタンを配置することが重要です。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/undo-and-redo#Platform-considerations)

 _visionOSに追加の考慮事項はありません。tvOSとwatchOSには対応していません。_

### [iOS/iPadOS](/jp/design/human-interface-guidelines/undo-and-redo#iOS-iPadOS)

**取り消し/やり直しの標準ジェスチャを定義し直さない。** 例えば、3本指のスワイプを使って取り消し/やり直しを開始できます。iPhoneをシェイクする方法もあります。すべての標準ジェスチャに言えることですが、アプリのインターフェイスでジェスチャを定義し直すと、ユーザの混乱を招き、アプリの操作が予測しにくくなるおそれがあります。

**取り消し/やり直しの対象となる操作を簡潔かつ正確に示す。** 取り消し/やり直しの通知タイトルには、「取り消す - 」または「やり直す - 」（末尾の空白を含む）というプレフィックスが自動的に追加されます。このプレフィックスのあとに、取り消し/やり直しの対象となる操作を示す追加の語句を表示する必要があります。例えば、「取り消す - 名前」や「やり直す - アドレスを変更」などの通知タイトルを作成できます。

### [macOS](/jp/design/human-interface-guidelines/undo-and-redo#macOS)

**取り消し/やり直しコマンドを「編集」メニューに配置し、標準のキーボードショートカットに対応させる。** Macユーザは、「編集」メニューの上部に取り消し/やり直しがあることを期待します。また、Command+ZとShift+Command+Zを使用して取り消しとやり直しをそれぞれ実行できることも期待します。
