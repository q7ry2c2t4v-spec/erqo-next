# 設定

アプリやゲームは何もしなくても動作することが期待されますが、ユーザのしたいことに合わせて体験をカスタマイズできることも高く評価されます。

![歯車のスケッチ。設定を示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/6257b9f6fbd58f3979c82060d7a3c373/patterns-settings-intro%402x.png)

すべてのAppleプラットフォームでは、システムが提供する設定アプリで、システムの全体的な見た目、ネットワーク接続、アカウントの詳細、アクセシビリティの要件、言語と地域の設定のようなオプションを調整できます。一部のプラットフォームでは、システムが提供する設定アプリに特定のアプリやゲームの設定を含めることもでき、多くの場合、そのアプリやゲームが場所の情報にアクセスしたり、マイクやカメラなどのデバイス機能を使用したり、通知、Siri、「検索」などのシステム機能と統合したりできるかどうかをユーザが調整することができます。

必要な場合は、アプリやゲームの中にカスタム設定領域を用意して、インターフェイスのスタイルや重要な動作など体験全体に影響する一般的な設定を提供することができます。特定のタスクのみに影響する設定を提供する必要がある場合は、タスク自体の中にこれらのオプションを含めることができるため、ユーザがカスタマイズのために体験を終了する必要はありません。

## [ベストプラクティス](/jp/design/human-interface-guidelines/settings#Best-practices)

**最も多くのユーザに最高の体験を届けられるようなデフォルト設定を提供する。** 例えば、ゲームが実行されているデバイスのパフォーマンス最大化は、ゲームの起動後にプレイヤーに選択させずに、自動的に行うことができます（デベロッパ向けのガイダンスは、[ゲームのグラフィックスパフォーマンスと設定を改善する](/documentation/Metal/improving-your-games-graphics-performance-and-settings)（英語）を参照してください）。適切なデフォルト設定を選択すれば、ユーザが何も調整しなくてもアプリやゲームを楽しめるようになる場合があります。

**提供する設定の数をできるだけ少なくする。** ユーザはアプリやゲームを制御したいと思っていますが、設定項目が多すぎると、体験が親しみにくく感じられるだけでなく、特定の設定を見つけるのも難しくなります。

**ユーザが期待する方法で設定を使用できるようにする。** 例えば、物理キーボードが接続されている場合は、アプリの設定を開くのに標準的なCommand＋カンマ（,）のキーボードショートカットが使われることが多いですが、ゲームではEscキーがよく使われます。

**設定でほかの方法で取得できる設定情報を求めない。** 例えば、ゲームでは、接続されたコントローラやアクセサリを自動的に検出でき、プレイヤーに識別してもらう必要はありません。アプリでは、ユーザが今ダークモードを使っているかどうかを検出することができます。

**ユーザのシステム全体の設定を尊重し、カスタム設定領域に重複した設定がないようにする。** ユーザは、システム提供の設定アプリを使用してアクセシビリティ調整、スクロール動作、認証方法のようなグローバルオプションを管理して、自分の選択がすべてのアプリやゲームに適用されることを期待しています。グローバルオプションのカスタムバージョンを設定領域に含めると、ユーザが混乱する可能性があります。システム全体の設定がこのアプリやゲームに適用されず、グローバル設定のカスタムバージョンを変更するとほかのアプリやゲームが影響を受けることが予期されるためです。

## [一般的な設定](/jp/design/human-interface-guidelines/settings#General-settings)

**カスタム設定領域には一般的であまり変更しない設定を配置する。** ユーザがアプリやゲームの設定領域を開くには作業を中断する必要があるため、頻繁に変更する必要のないオプションを含めることをおすすめします。例えば、アプリの場合はウインドウ設定を調整するオプションの一覧を配置します。ゲームの場合は、重要な動作やキーボードマッピングをプレイヤーが指定できるようにします。そしてアプリとゲームの両方で、ユーザのアカウントに関するオプションを提供します。

## [タスク固有のオプション](/jp/design/human-interface-guidelines/settings#Task-specific-options)

**可能な場合は、ユーザが設定領域に移動しなくてもタスク固有のオプションを変更できるようにする。** 例えば、ユーザが現在のビューの一部分を表示/非表示にしたり、項目のコレクションを並べ替えたり、リストをフィルタリングしたりするような調整ができる場合は、その設定が反映される画面でオプションを提供すると、見つけやすくて便利です。このようなオプションを別の設定領域に配置すると、オプションが現在のタスクから切り離され、ユーザは自分のタスクを中断して設定を調整しなければならず、多くの場合、タスクを再開するまでは反映されたかどうかが分かりません。

備考

ゲームでは、プレイヤーが特定のタスクへのアプローチを調整するときは、設定オプションの変更としてではなく、ゲームプレイの一部として行う傾向があります。

## [システム設定](/jp/design/human-interface-guidelines/settings#System-settings)

**めったに変更しないオプションのみをシステム提供の設定アプリに追加する。** また、アプリやゲームの設定をシステム提供の設定アプリに追加するのが適切である場合は、インターフェイスから直接設定を開くボタンを配置することを検討してください。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/settings#Platform-considerations)

 _iOS、iPadOS、tvOS、visionOSに追加の考慮事項はありません。_

### [macOS](/jp/design/human-interface-guidelines/settings#macOS)

ユーザがアプリやゲームのアプリメニューから設定項目を選択すると、カスタム設定ウインドウが開きます。通常、カスタム設定ウインドウには、 _パネル_ と呼ばれるビューを切り替えるためのボタンを含むツールバーがあります。各パネルには、関連する一連の設定が含まれています。

**設定項目は[アプリメニュー](/jp/design/human-interface-guidelines/the-menu-bar#App-menu)に含める。**ウインドウのツールバーに設定ボタンを追加しないでください。ユーザが頻繁に使用する重要なコマンドを配置する領域が減るためです。書類レベルのオプションを提供する場合は、アプリの[「ファイル」メニュー](/jp/design/human-interface-guidelines/the-menu-bar#File-menu)にその項目を追加します。

**設定ウインドウの最小化/最大化ボタンを暗くする。** 標準のCommand+カンマ（,）キーボードコマンドを使用してカスタム設定ウインドウを素早く開くことができるため、ウインドウをDockにしまっておく必要はありません。また、設定ウインドウは現在のパネルのサイズに合わせて調整されるため、さらに表示するためにウインドウを広げる必要もありません。

**設定ウインドウでは、表示されたままのカスタマイズできないツールバーを使用して、アクティブなツールバーボタンを常に示すようにする。** 設定ウインドウのツールバーは、ユーザがカスタマイズできる領域を示し、そのような領域間を移動できるようにします。固定された設定インターフェイスは、ユーザが必要な項目を見つけるのに役立ちます。

**現在表示されているパネルに合わせてウインドウのタイトルをアップデートする。** 設定ウインドウに複数のパネルがない場合は、「 _アプリ名_ の設定」というタイトルを使用します。

**最後に表示したパネルを復元する。** ユーザは通常、関連する設定を何度も調整するため、設定ウインドウを開いたときに前回使用したパネルが表示されると便利です。

### [watchOS](/jp/design/human-interface-guidelines/settings#watchOS)

watchOSでは、アプリやゲームのカスタム設定がシステム提供の設定アプリに追加されることはありません。代替手段として、メインビューの下部に少数の必須オプションを配置するか、「詳細」メニューでオブジェクトを再設定できるようにすることを検討してください。
