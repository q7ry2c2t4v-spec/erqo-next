# トップシェルフ

Apple TVのホーム画面にはトップシェルフと呼ばれる領域があります。魅力的でリッチなコンテンツが表示されるこの領域から、Dock内のよく使うアプリにアクセスすることもできます。

![図案化されたApple TVの画面。Apple TVアプリの列の上にメディアのプレビューのリストが横並びで表示されています。画像は全体が、6色のオリジナルAppleロゴを連想させる赤色になっています。](https://docs-assets.developer.apple.com/published/efbe0358fd4d5529cfe2c39594c8866a/components-top-shelf-intro%402x.png)

アプリがフルスクリーンのトップシェルフに対応している場合、ユーザは、複数のフルスクリーンコンテンツビューをスワイプで切り替えたり、トレイラーやプレビューを再生したり、コンテンツの詳細情報を見たりできます。

トップシェルフは、新作、注目、おすすめのコンテンツを紹介する場所ですが、ユーザがそこからアプリやゲームに直接ジャンプして視聴やプレイもできるユニークなコンポーネントです。例えばユーザがDockでApple TVを選択すると、すぐにDockが隠れて、即座にフルスクリーンのプレビューが再生されます。ユーザは1つ目の番組のプレビューを視聴している間も、スワイプでほかの注目の番組のプレビューを切り替えながらチェックできます。気になるプレビューがあったら、そこで「再生」や「詳細情報」を選択できます。

Dockでアプリが選択されたときに表示されるトップシェルフの体験を魅力的なものにできるよう、システムではさまざまなレイアウトテンプレートが定義されています。コンテンツの配置に役立つこれらのテンプレートは、[Appleデザインリソース](https://developer.apple.com/design/resources/#tvos-apps)からダウンロードできます。

## [ベストプラクティス](/jp/design/human-interface-guidelines/top-shelf#Best-practices)

**コンテンツに直接ジャンプできるようにする。** トップシェルフは、ユーザが最も関心のあるコンテンツにたどり着くための近道として機能します。システムの2つの標準レイアウトテンプレートである[カルーセルアクション](/jp/design/human-interface-guidelines/top-shelf#Carousel-actions)と[カルーセルディテール](/jp/design/human-interface-guidelines/top-shelf#Carousel-details)には、それぞれ2つのボタンがデフォルトで含まれています。再生を開始するためのプライマリボタンと、アプリを開いてコンテンツの詳細情報を表示するための「詳細情報」ボタンです。

**新しいコンテンツに注目させる。** 例えば、新作や新着エピソードを紹介し、公開予定の映画や番組に注目させます。ユーザがすでに購入、レンタル、視聴したコンテンツを宣伝しないようにしてください。

**ユーザのお気に入りのコンテンツに合わせてパーソナライズする。** ユーザは、よく使うアプリをトップシェルフに置くものです。個々のユーザに合わせたおすすめをトップシェルフのコンテンツに表示し、メディアの再生を再開したりプレイ中のゲームにすぐに戻ったりできるようすれば、体験をパーソナライズできます。

**広告や価格を表示するのは避ける。** ユーザはアプリを利用したいからトップシェルフに追加するので、アプリの広告をいくつも見せられるのは心よい体験とは言えません。購入可能なコンテンツをトップシェルフで見せることもできますが、それよりもむしろ楽しい新作コンテンツの紹介に注力しましょう。価格を表示するのはユーザが関心を示した場合に限定することをおすすめします。

**ユーザの注意を引き付け、もっと見たいと思わせるような魅力にあふれたコンテンツを紹介する。** 必要であれば静止画像も使えますが、通常、最新作や高評価のコンテンツを紹介してくれる、魅力的でダイナミックなトップシェルフの方が好まれます。こうした体験を提供するには、トップシェルフに表示する[レイヤード画像](/jp/design/human-interface-guidelines/images#Layered-images)を作成することをおすすめします。

**推奨されるフルスクリーンコンテンツを提供しない場合、代用できる静止画像を1点以上追加する。** Dockでフォーカスされたアプリにフルスクリーンコンテンツがない場合は静止画像が表示されます。画像は回転しながらブラーが適用され、アスペクト比16:9で幅1920ピクセルになるように調整されます。以下のガイドに従ってください。

画像サイズ  
---  
2320x720 pt（2320x720 px @1x、4640x1440 px @2x）  
  
**静止画像をインタラクティブに操作できるような印象を与えない。** トップシェルフの静止画像はフォーカスできません。インタラクティブに操作できると誤解されないようにしましょう。

## [動的なレイアウト](/jp/design/human-interface-guidelines/top-shelf#Dynamic-layouts)

動的なトップシェルフの画像は以下の複数の方法で表示できます:

  * 2つのボタンと詳細情報（任意）を含むフルスクリーンのビデオや画像をカルーセル形式で見せる方法

  * フォーカス可能なコンテンツを1行に並べる方法

  * スクロールするバナーのセットを表示する方法

### [カルーセルアクション](/jp/design/human-interface-guidelines/top-shelf#Carousel-actions)

カルーセルアクションは、フルスクリーンの動画や画像をメインで見せ、続きや詳細を見るための数個のコントロールを控えめに添えるレイアウトスタイルです。ユーザがある程度認識しているコンテンツを紹介するのに特に適しています。例えば、ユーザが生成した写真や、シリーズや番組の新しいコンテンツなど、ユーザが好きそうなコンテンツの表示に適しています。

**タイトルを含める。** 番組や映画のタイトル、写真アルバムのタイトルなど、簡潔なタイトルを含めてください。必要であれば短いサブタイトルを表示できます。例えば、写真アルバムなら日付の範囲、エピソードならその番組名などをサブタイトルにできます。

### [詳細カルーセル](/jp/design/human-interface-guidelines/top-shelf#Carousel-details)

カルーセルアクションを拡張したレイアウトスタイルで、コンテンツに関する情報を含めることができます。例えば、コンテンツを選ぶ際の判断材料になりそうなメタデータ（あらすじやキャストなど）の情報を提供できます。

**現在再生中のコンテンツが分かるタイトルを表示する。** コンテンツのタイトルは、ぱっと見て目に入りやすい画面の上部付近に表示されます。タイトルの上に短い語句やアプリの属性情報（例: 「 _[アプリ名]_ のおすすめ」）を含めることもできます。

### [セクションコンテンツ行](/jp/design/human-interface-guidelines/top-shelf#Sectioned-content-row)

セクションコンテンツをラベル付きの単一行に並べて見せるレイアウトスタイルです。最近閲覧したコンテンツや新しいコンテンツ、よく使うコンテンツを強調するのに適しています。行コンテンツはフォーカス可能で、素早くスクロールすることができます。コンテンツにフォーカスするとラベルが表示され、リモコンのタッチサーフェスに触れて少し動かすとフォーカス中の画像が動きます。セクションコンテンツ行に複数のラベルが表示されるように設定することもできます。

**行を埋められるだけのコンテンツを用意する。** セクションコンテンツ行には、少なくとも画面の幅全体を埋められるだけの画像を読み込みましょう。プラットフォーム間の一貫性を高め、追加のコンテキストを提供できるようにラベルを1つ以上含めてください。

セクションコンテンツ行では以下の画像サイズを使用できます。

#### [スクロールするインセットバナー](/jp/design/human-interface-guidelines/top-shelf#Scrolling-inset-banner)

ほぼ画面の幅全体に広がる大きな画像を並べるレイアウトです。いずれかのバナーにフォーカスが当たるまで、バナーはプリセットのタイマーに従って自動的にスクロールします。最後の画像に達したらまた最初の画像に戻るというように、表示は循環します。

バナーにフォーカスが当たっているときにリモコンのタッチサーフェス上で小さな円を描くように指を動かすと、フォーカスエフェクトが実行され、そのバナーが動いてライティングエフェクトがかかります。バナーにレイヤード画像が含まれている場合は3Dエフェクトもかかります。タッチサーフェスでスワイプすると、並んでいる次または前のバナーに移動します。人気の新作映画など、魅力的なリッチコンテンツを紹介するときにはこのスタイルを使いましょう。

**3～8点の画像を含める。** スクロールするバナーを効果を引き出すには、最低でも3点の画像を含めることをおすすめします。画像が8点より多いと、目的の画像に移動しづらくなる可能性があります。

**テキストが必要な場合は画像に追加する。** このレイアウトスタイルではコンテンツの下にラベルが表示されないため、テキストはすべて画像自体に含める必要があります。レイヤード画像の場合は、一番上にテキスト専用のレイヤーを配置してテキストが浮かび上がるようにすることを検討してください。[VoiceOver](/jp/design/human-interface-guidelines/voiceover)で読み上げられるように、テキストを画像のアクセシビリティラベルにも追加します。

![外側の特大幅の長方形の中に、少し小さい長方形があり、その中にさらに小さい長方形を表示している図。一番外側の長方形は実際のサイズ、真ん中の長方形はセーフゾーン（表示可能な領域）、一番内側の長方形はフォーカスが当たっていないときのサイズを表しています。](https://docs-assets.developer.apple.com/published/ef2d1c53a0f50659306ab53ce0b7f9e7/icons-and-images-content-layout-extra-wide%402x.png)
