# スキル一覧

スキルの詳細な手順は各 SKILL.md に記載されている。ここでは「いつ何を使うか」の判断基準を示す。

## メインフローのスキル

### /dsgn — 対話・設計

- **いつ:** プロジェクト初期に1度
- **目的:** 大枠の機能セクション一覧を決める
- **成果物:** `.libs/design/project-spec/`
- **注意:** 各セクションの詳細には踏み込まない

### /tp SECTION — タスク計画

- **いつ:** セクションに着手する時
- **目的:** 詳細設計 + タスク分解 + 識別コード・依存関係付与
- **成果物:** `.libs/design/<section>/`
- **前提:** /dsgn で対象セクションが定義済みであること
- **コンテキスト収集:** `core/index.py collect` で既存の関連ページを自動収集

### /codi ID-CODE — 実装パイプライン

- **いつ:** タスクを実装する時
- **目的:** 5ステップの自動化パイプライン
- **成果物:** 実装コード + 記録ページ + ユーザーガイド + コミット
- **前提:** /tp で対象タスクの識別コードが付与済みであること
- **状態管理:** `core/state.py` がパイプライン状態を管理。中断後の再開可能
- **依存チェック:** 開始時に依存タスクの完了状態を自動チェック

## 補助スキル

### /rsrc TOPIC — ウェブリサーチ

- **いつ:** 実装に必要な技術情報を事前に調べたい時
- **目的:** ウェブリサーチを行い、結果を .libs に保存する
- **成果物:** `.libs/research/<topic>/` にリサーチページ
- **特徴:** タグ付きで保存されるため `/tp` の index.py collect から自動参照される

### /gh — プロジェクト Git 管理

- **いつ:** 変更をコミット・プッシュしたい時
- **目的:** プロジェクトの git 操作を安全に実行する
- **成果物:** git コミット（+ プッシュ）
- **特徴:** 差分確認 → コミットメッセージ生成 → ユーザー承認 → 実行。`/codi` ステップ5 からも呼ばれる
- **スクリプト:** `skills/gh/handler.py`（check / commit / push / full）

### /ruru — プロジェクトルール管理

- **いつ:** ルールを決めた時
- **目的:** プロジェクトルールの登録・更新
- **成果物:** `.libs/rules/` + CLAUDE.md に @import 追加
- **特徴:** ルール内容からスコープ（`適用: ui/api/all`）を自動推定。load.py のスコープフィルタリングと連携

### /mmy — セッションログ

- **いつ:** セッション終了時
- **目的:** セッション内容をレポートとして記録
- **成果物:** `.libs/session-logs/YYYY-MM/slog-YYYYMMDD-NNN.md`
- **特徴:** 保存前に品質セルフチェック（復元可能性・決定事項の具体性・未完了の実行可能性）

### /ret — セッション復元

- **いつ:** 前回の続きをしたい時、キーワードが不明な時
- **目的:** 直近セッションログから前回コンテキストを取得
- **成果物:** なし（読むだけ）
- **特徴:** セッションログのタグから `index.py collect` を自動実行し、関連コンテキストを拡充

### /fb — フィードバック送信

- **いつ:** erqo-nxt への改善提案・エラー報告を送りたい時
- **目的:** プロジェクトから erqo-nxt 本体へフィードバックを送信する
- **成果物:** `.claude/state/feedback/` に JSON レポート
- **スクリプト:** `skills/fb/handler.py`

## スキルの選び方

```
セッション開始
  ├── 前回の続き → /ret
  ├── 新規プロジェクト → /dsgn
  ├── 技術情報を調べたい → /rsrc TOPIC
  ├── セクション着手 → /tp SECTION
  ├── タスク実装 → /codi ID-CODE
  ├── 変更をコミット → /gh
  ├── ルール追加 → /ruru
  ├── フィードバック送信 → /fb
  └── セッション終了 → /mmy
```
