# セッション管理

## セッション開始（自動）

SessionStart Hook により、セッション開始時に `core/session.py` が自動実行される。

### 視点宣言ゲート（必須・前置）

通常モード・compact モードの両方で、番号付き項目より前に「視点宣言ゲート」ブロックが先頭注入される。

- `IS_SOURCE` の値から「本元 OS」「プロジェクト OS」のどちらかを機械的に判定して提示する
- エル子は最初の応答の冒頭で、以下を必ず宣言してから作業に入る:
  - 視点: 本元 OS / プロジェクト OS
  - 依頼のスコープ: 本元 / プロジェクト / 共通
  - 根拠: その判断の理由
- 宣言が済むまで一切の実作業には入らない

**目的:** `specs/07-scopes.md` の 3 スコープ判断を毎セッション能動的に照合させ、視点の取り違えによるシステム機能間の予期しない副作用を未然に防ぐ。受動的にコンテキストを読むだけでは取り違えが起きうるため、能動的な宣言を強制する。

### 通常モード（startup / resume）

以下の7項目がコンテキストに注入される:

1. **プロジェクト名** — PROJECT_ROOT.name を表示
2. **インデックス再生成** — `index.py reindex` で _index.json を最新化
3. **バージョンチェック** — VERSION ファイル vs GitHub API タグ比較（1日1回制限）
4. **git 同期チェック** — リモートとの差分を確認。安全時は自動 `pull --ff-only`
5. **中断パイプライン** — `/codi` が途中で中断された場合、タスク ID とステップ番号を表示
6. **進行中タスク** — TP ページから「未着手」「進行中」のタスクを抽出
7. **直近セッションログ** — 最新の `/wrap` レポートの先頭30行

補助的に、以下も裏で実行・注入される（本線 7 項目の間に差し込まれる）:

- **研究ノート自動同期** — `research_sync.auto_pull` で `.libs/research/` を最新化（clone 済みの場合のみ、失敗は警告のみで続行）
- **研究ノート未 clone 警告** — `.libs/research/` が未 clone のとき、復旧コマンド（本元: `python core/dev.py setup` / プロジェクト: `python .nxt-core/core/install.py --update`）を表示。PC 乗り換え後の初回セットアップ忘れ対策
- **ローカル specs 同期チェック** — プル子向け（`check_local_specs_sync`）

### compact モード（コンパクション後）

コンテキストが圧縮された後に発火する。最小限の3項目を再注入:

1. **中断パイプライン** — 完了済みステップも表示
2. **進行中タスク**
3. **直近決定事項** — 最新セッションログの「決定事項」セクションを `page_parser.parse_sections()` で構造的に抽出

compact モードでも補助的に **研究ノート未 clone 警告** を冒頭に出す（未 clone を見落とさないため）。

## 中断復旧

`/codi` パイプラインがセッション中断で途切れた場合の復旧。

### 検知

`session.py` が `.claude/state/` をスキャンし、`codi_*.json` が残っていれば中断と判定。コンテキストに「/codi TASK-ID で再開できます」と注入する。

### 復旧フロー

`/codi TASK-ID` を再実行すると:

1. `state.py init` が既存の状態ファイルを検出
2. `completed_steps` に含まれるステップをスキップ
3. 中断箇所から再開

### 状態ファイル

```json
{
  "task_id": "AUTH-SESSION-FLOW",
  "current_step": 3,
  "completed_steps": ["prepare", "implementation"],
  "started_at": "2026-04-02T10:30:00+09:00",
  "last_update": "2026-04-02T10:45:00+09:00"
}
```

パイプライン完了時に自動削除される。

## git 同期チェック

セッション開始時にリモートとの同期状態をチェックする。

| 状態 | 動作 |
|---|---|
| 同期済み | 何もしない |
| リモートが先行 + ローカル変更なし | 自動 `pull --ff-only` |
| リモートが先行 + ローカル変更あり | 通知のみ（自動 pull しない） |
| ローカルが先行 or 分岐 | 何もしない |

IS_SOURCE（本体リポジトリ）ではスキップする。

## バージョンチェック

`NXT_ROOT/VERSION` と GitHub API の最新タグを比較する。

- `git ls-remote --tags` で最新タグを取得
- 1日1回制限（`_version_check.json` にタイムスタンプ保存）
- 新バージョンがあれば通知
- ネットワーク不通時はタイムアウト5秒で静かにスキップ

## 手動復元（/ret）

セッション開始時にキーワードが不明な場合、`/ret` で直近のセッションログから前回コンテキストを取得する。

- 直近2〜3件のセッションログを読む
- 前回の作業内容・決定事項・未完了事項を要約して伝える
- セッションログのタグから `index.py collect` を自動実行し、関連コンテキストを拡充

## セッション記録（/wrap）

セッション終了時に `/wrap` でセッション内容のレポート + git コミットを 1 コマンドで実行する。

- 保存先: `.libs/session-logs/YYYY-MM/slog-YYYYMMDD-NNN.md`
- 4セクション: 概要、作業内容、決定事項、未完了・次回
- 保存前に品質セルフチェック（復元可能性・決定事項の具体性・未完了の実行可能性）
- 作成後に `index.py reindex` でインデックスを更新
- 同じコマンドの中で `git_ops.py commit` を呼び、ログを含む変更を 1 コミットにまとめる
- 次回セッションの session.py や /ret から参照される
