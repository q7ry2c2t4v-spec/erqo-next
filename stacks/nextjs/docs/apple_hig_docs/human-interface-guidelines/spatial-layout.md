# 空間レイアウト

空間レイアウトのテクニックを使うと、Apple Vision Proの無限のキャンバスを有効活用し、ユーザに負担をかけずに魅力的にコンテンツを提示することができます。

![立体的なレイアウトであることを示す、X軸、Y軸、Z軸のスケッチ。画像の上に長方形と円形のグリッド線が重ねられており、画像全体が6色の初代Appleロゴの黄色を思わせる黄色みを帯びています。](https://docs-assets.developer.apple.com/published/e13737927c465ae264094aa019129252/foundations-spatial-layout-intro%402x.png)

## [視野](/jp/design/human-interface-guidelines/spatial-layout#Field-of-view)

ユーザの _視野_ とは、頭を動かさずに見える空間のことです。Apple Vision Proを装着しているときの各ユーザの視野の広さは、ライドシールドの設定や個々の周辺視力の程度などに左右されます。

![visionOSに表示された空のアプリウインドウのスクリーンショット。30度、60度、90度の視野を示す一連の同心円が画像の上に重ねて表示されています。](https://docs-assets.developer.apple.com/published/88086621da558b375ed5ef8ea0002283/visionos-field-of-view-layout%402x.png)

重要

システムはユーザの視野に関する情報を提供しません。

**重要なコンテンツは視野の中心に配置する。** デフォルトでは、アプリはユーザの正面で起動し、ユーザの視野の中に配置されます。イマーシブ体験では、ユーザの注意を重要なコンテンツに向けるために、そのようなコンテンツを常に中央に配置するようにし、周辺に邪魔になる動きや、コントラストの高い明るいオブジェクトを表示しないようにします。

以下のカスタムコントロールを使用できます 

このビデオコンテンツの説明: Apple Vision Proを装着し、背筋を伸ばして椅子に座っている人のアニメーション。visionOSのアプリウインドウを表す正方形が視野の中心にあり、装着者はそれを正面から見ています。装着者の目からウインドウの中心に向かう点線がアニメーションされています。 

再生 

以下のカスタムコントロールを使用できます 

このビデオコンテンツの説明: Apple Vision Proを装着し、椅子にもたれて座っている人のアニメーション。装着者はvisionOSのアプリウインドウを表す正方形を見ています。アプリウインドウは宙に浮き、装着者との距離が近くなっています。また、視野の中心に来るように装着者の方に傾いています。装着者の目からウインドウの中心に向かう点線がアニメーションされています。 

再生 

**装着者の頭にコンテンツをアンカリングしない。** アプリは視野の中に留めておくのが基本ですが、常にユーザの正面に固定されるようにコンテンツをアンカリングすると、ユーザが「ここから抜け出せない」「コンテンツに縛られている」「不快」などと感じてしまうかもしれません。コンテンツによってパススルーの大部分が覆われ、周囲の環境が不安定になったように感じるような場合はなおさらです。ユーザが自然に周囲を見渡して、ほかの位置にあるほかのオブジェクトも見ることができるように、コンテンツはユーザの空間にアンカリングしてください。

## [奥行き](/jp/design/human-interface-guidelines/spatial-layout#Depth)

人は、物体との距離や、物体同士の前後関係、影などの視覚情報をもとに周囲の環境を把握します。Apple Vision Proのシステムは、色温度や反射、影などの視覚エフェクトを自動的に使用して、仮想コンテンツに奥行き感を与えます。ユーザが空間内で仮想オブジェクトを移動したとき、またはユーザとオブジェクトとの相対的な位置関係が変わったときには、視覚エフェクトによってオブジェクトの見かけ上の奥行きが変化することで、より現実らしく感じられるようになっています。

オブジェクトはどのような角度からでも見ることができるので、標準ウインドウも含めてインターフェイス全体にわずかな奥行きを与えれば、より自然な見た目のインターフェイスになります。SwiftUIを使用する場合は、2Dウインドウのビューに自動的に視覚エフェクトが追加され、奥行き感が与えられます。デベロッパ向けのガイダンスは、[アプリに3Dコンテンツを追加する](/documentation/visionOS/adding-3d-content-to-your-app)を参照してください。

![visionOSの2Dの「メモ」ウインドウのスクリーンショット。ビューの末尾側で「Nature Walks」というタイトルのメモが開いていて、葉のスケッチに手書きのテキストの説明が添えられています。](https://docs-assets.developer.apple.com/published/2b07a7f22124deaea6c2ce31a93d8833/visionos-spatial-layout-2d-window%402x.png)

コンテンツにさらに奥行きを出す必要がある場合は、RealityKitを使って3Dオブジェクトを作成します（デベロッパ向けのガイダンスは、[RealityKit](/documentation/RealityKit)を参照してください）。3Dオブジェクトはどこにでも表示できますが、3Dコンテンツを表示する _ボリューム_ というコンポーネントを使うこともできます。ボリュームはウインドウに似ていますが、フレームは表示されません。ガイダンスは[visionOSのボリューム](/jp/design/human-interface-guidelines/windows#visionOS-volumes)を参照してください。

以下のカスタムコントロールを使用できます 

このビデオコンテンツの説明: visionOSのボリューム内に人工衛星の3Dモデルが表示されている録画。装着者が人工衛星に近づいてその向きを操作すると、装着者の位置と人工衛星の角度に基づいて光の反射が変化します。 

再生 

**コンテンツの奥行きを正確に伝える視覚情報を提供する。** 視覚情報がなかったり、視覚情報が現実世界での体験と矛盾していたりすると、視覚的な不快感が生じる可能性があります。

**奥行きで階層を伝える。** 奥行きは、オブジェクトと周囲のコンテンツとの間に視覚的な差を付け、オブジェクトをより目立たせるのに役立ちます。また、人間は一般的に奥行きの変化に敏感なので、例えばウインドウの上にシートを表示させれば、ウインドウがZ軸方向に後退し、シートが手前に出て視覚的に目立つようになります。

**テキストには基本的に奥行きを付けない。** バックグラウンドから浮いたように見えるテキストは読みにくく、ユーザの読む速度が落ちてしまいます。視覚的な不快感が生じる場合もあります。

**奥行きに意味を持たせる。** 基本的に、奥行きは明快にしたりユーザを楽しませたりするために使うものです。すべてに奥行きを出す必要はありません。デザインに奥行きを出す場合は、オブジェクトのサイズと相対的な重要度を考えましょう。タブバーやツールバーをウインドウから浮かび上がらせるなど、大きくて重要なアプリ要素を視覚的に区別するのには奥行きが効果的ですが、小さいオブジェクトには同じような効果は期待できないかもしれません。例えば、ボタンのシンボルに奥行きを与えてバックグラウンドから浮かび上がらせると、かえって読みにくく、使いにくくなってしまう可能性があります。また、アプリ内で何種類も奥行きを使う場合は使用頻度に注意しましょう。奥行きがそろっていない場合、ユーザは奥行きが変わるたびに目のピントを調整しなければならず、それをあちこちで素早く行うと疲れてしまう可能性があります。

## [スケール](/jp/design/human-interface-guidelines/spatial-layout#Scale)

visionOSでは、見かけの奥行きを維持しつつ使いやすさを最適化するために、2つのタイプのスケールが定義されています。

1つは、自分との距離にかかわらずコンテンツを快適に読んで操作できるようにするための _ダイナミックスケール_ です。具体的には、ウインドウが装着者から離れるとウインドウのスケールが自動的に大きくなり、近づくとスケールが小さくなることで、どの距離にあるときでも同じサイズに見えるようになっています。

以下のカスタムコントロールを使用できます 

このビデオコンテンツの説明: アプリウインドウを表す正方形が3D空間に表示されているアニメーション。正方形が最初の位置から平面に沿って後退し、移動と同時にサイズが動的に大きくなります。比較しやすいように、最初の位置を表すフレームが表示されたままになっています。移動の完了後、環境全体が回転し、装着者のアングルからはウインドウのサイズが常に一定であることが示されます。 

再生 

もう1つの _固定スケール_ では、装着者との距離にかかわらずオブジェクトのスケールが一定に保たれます。物理環境にある物体は近くにあるときよりも遠くにあるときの方が小さく見えます。これと同様に、固定スケールのオブジェクトは、Z軸方向に後退すると小さくなったように見えます。

以下のカスタムコントロールを使用できます 

このビデオコンテンツの説明: アプリウインドウを表す正方形が3D空間に表示されているアニメーション。正方形が最初の位置から平面に沿って後退し、移動と同時に小さくなります。比較しやすいように、最初の位置を表すフレームが表示されたままになっています。移動の完了後、環境全体が回転し、装着者のアングルからはウインドウが遠くに後退したように見えることが示されます。 

再生 

ほかのプラットフォームでは、ポイントはピクセル数として定義されていて1ポイントが何ピクセルになるかは2Dディスプレイの[解像度](/jp/design/human-interface-guidelines/images#Resolution)によって異なりますが、visionOSの場合は、ダイナミックスケールと奥行き感を実現するために、ポイントが角度として定義されます。

**仮想オブジェクトを現実世界の物体のように見せたい場合は固定スケールの使用を検討する。** 例えば、ユーザが自分の空間に表示したときによりリアルに見えるように、取り扱っている商品の実際のサイズを維持したい場合などです。インタラクティブなコンテンツの場合は、近くにあっても遠くにあっても使いやすい大きさである必要があるので、固定スケールの使用は控えめにし、それを必要とする非インタラクティブなオブジェクトのみに留めてください。

## [ベストプラクティス](/jp/design/human-interface-guidelines/spatial-layout#Best-practices)

**表示するウインドウを多くしすぎない。** ウインドウが多すぎると周囲の環境が隠れ、圧迫感や束縛感、あるいは不快感を与えてしまうかもしれません。また、多くのウインドウを動かすことになるため、アプリの位置調整も面倒になります。

**標準の間接的ジェスチャを優先する。** _間接的_ ジェスチャは、手を視野の中に持ってこなくても実行できます。反対に、 _ダイレクト_ ジェスチャを行うには自分の指で仮想オブジェクトに触れなければなりません。特に、オブジェクトが目線の高さ以上にある場合には、これが疲れを生じさせる原因になります。visionOSのユーザは、すでに知っている標準のジェスチャを行うときには間接的ジェスチャを使います。間接的ジェスチャを優先すれば、注視するオブジェクトがどのような距離にあっても、それを使って操作することができます。直接的ジェスチャに対応する場合は、短時間だけ詳しく見たり操作したりしてほしい近距離のオブジェクトだけに留めることを検討してください。ガイダンスは[ジェスチャ＞visionOS](/jp/design/human-interface-guidelines/gestures#visionOS)を参照してください。

**ウインドウを視野の中心に置き直す機能はDigital Crownに任せる。** ユーザは、移動したり首を回したりしたときに、コンテンツの表示位置を変えたくなることがあるかもしれません。そうした場合には、[Digital Crown](/jp/design/human-interface-guidelines/digital-crown)を押すことでコンテンツを自分の正面に置き直すことができます。このアクションにアプリ側で対応する必要はありません。

**注視しやすいように、インタラクティブなコンポーネントの周囲には十分なスペースを設ける。** ユーザがインタラクティブな要素を見つめると、目的の要素を判別できたかどうかを確かめられるように視覚的なホバーエフェクトが表示されます。インタラクティブなコンテンツを楽に注視できるようにし、かつ、ホバーエフェクトがほかのコンテンツにかからないようにするには、インタラクティブなコンテンツの周りに十分なスペースを確保することが非常に重要になります。例えば、複数の通常サイズの[ボタン](/jp/design/human-interface-guidelines/buttons#visionOS)の場合は、中心間の距離が60ポイント以上、ボタン間のスペースが16ポイント以上になるように配置してください。また、コントロールがほかのインタラクティブな要素やビューと重ならないようにしてください。これは、1つの要素を選択するのが難しくなる場合があるためです。

**アプリを使うのに必要な身体の動きを最小限にするかゼロにする。** 体験を楽しむにはある程度の身体の動きが不可欠というような場合を除き、誰もがその場から動かずに楽しめるようにしましょう。

**大きなイマーシブ体験を配置するときは床を利用する。** 床を起点にして大きく広がるコンテンツがイマーシブ体験に含まれている場合は、フラットな水平平面を使って配置しましょう。この水平面と床の位置を合わせれば、コンテンツを周囲の環境と継ぎ目なく融合させ、より直感的な体験を実現することができます。

visionOSのウインドウとボリュームについて詳しくは、[ウインドウ＞visionOS](/jp/design/human-interface-guidelines/windows#visionOS)を参照してください。ウインドウ内でのコンテンツの配置については、[レイアウト＞visionOS](/jp/design/human-interface-guidelines/layout#visionOS)のガイダンスを参照してください。
