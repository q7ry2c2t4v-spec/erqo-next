# CarPlay

CarPlayでは、経路案内、電話、メッセージの送受信、音楽の再生を、運転に集中しながら車載ディスプレイ上で実行できます。

![CarePlayアイコンのスケッチ。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせる青色になっています。](https://docs-assets.developer.apple.com/published/7a06c1731ea29d9887681b2f2af491fd/technologies-CarPlay-intro%402x.png)

ユーザは、ほかのアプリと同じようにCarPlayアプリをApp StoreからiPhoneにダウンロードしてインストールします。iPhoneを車に接続すると、インストールしたCarPlayアプリのアプリアイコンがCarPlayのホーム画面に表示されます。

CarPlayは、ドライバーが運転中に使用することを前提に設計されています。CarPlayアプリを設計するときは、この状況を念頭に置き、ユーザが最小限の操作で素早くタスクを実行できる機能を提供してください。

CarPlayアプリのインターフェイスを作成する際には、オーディオ、コミュニケーション、ナビゲーション、給油など、開発するアプリの種類に応じてシステムで定義されたテンプレートを使用します。各テンプレートにおいて、アプリがコンテンツを提供することで、iOSがそれをCarPlayでレンダリングします。UIコンポーネントの表示と車とのインターフェイスの処理はシステムによって行われるため、画面解像度の違いに応じてレイアウトを調整したり、さまざまなハードウェアタイプ（タッチスクリーン、ノブ、タッチパッドなど）からの入力を管理したりする必要はありません。

各種CarPlayアプリの作成方法とシステムで定義されたテンプレートの使用方法については、[CarPlayアプリプログラミングガイド](https://developer.apple.com/carplay/documentation/CarPlay-App-Programming-Guide.pdf)を参照してください。以下の一般的なデザインガイドラインは、すべての種類のCarPlayアプリに適用されます。

## [iPhoneの操作](/jp/design/human-interface-guidelines/carplay#iPhone-interactions)

CarPlayでは、接続したiPhoneの対応アプリが車載ディスプレイに表示され、運転中に使用できるように最適化されたシンプルなインターフェイスが適用されます。

**CarPlayの使用中はiPhoneでアプリを操作させない。** アプリの操作は車載のコントロールとディスプレイで行う必要があります。iPhoneでアプリを設定する必要がある場合は、必ずユーザが発車前に行うようにしてください。

**接続したiPhoneから入力しないとCarPlayの操作が受け付けられないような状況を避ける。** ユーザがiPhoneをバッグやトランクに入れるなどして運転中にiPhoneにアクセスできなくても、アプリが機能する必要があります。接続したiPhoneで問題を解決する必要がある場合は、ユーザが停車後に行えるようにしてください。

**ユーザがiPhoneのロックを解除しなくてもアプリが動作するようにする。** ほとんどのユーザはiPhoneをロックしたままCarPlayを使用するため、CarPlayアプリで提供する機能がこのシナリオで期待通りに動作するようにしてください。

### [オーディオ](/jp/design/human-interface-guidelines/carplay#Audio)

CarPlayでは、アプリがほかのオーディオソース（車載ラジオやナビゲーションシステムの音声メッセージなど）と共存することに注意してください。オーディオがアプリ体験の主要な要素かどうかに関係なく、ユーザがオーディオにどのような動作を期待しているかを知り、その期待に応える必要があります。

**ユーザが再生を開始するタイミングを選べるようにする。** 基本的に、アプリの目的が単一のオーディオソースの再生である場合や以前に中断されたオーディオを再開する場合を除き、再生を自動的に開始することは避けてください。また、オーディオセッションを開始するとほかのオーディオソース（車載ラジオなど）の音が消えるため、オーディオを実際に再生できる状態になるまではセッションを開始しないでください。

**オーディオが十分に読み込まれた直後に再生を開始する。** バッファリングやネットワークの状態によっては、ユーザがオーディオを選択してから再生が開始されるまでに数秒かかる場合があります。オーディオ再生の準備ができたことをアプリがシステムに伝えるまでは、選択されたオーディオがハイライトされたまま、回転するアクティビティインジケータが表示されます。

**オーディオ再生の準備ができたら「再生中」画面を表示する。** 説明用の情報の読み込み完了を待たずに再生を始めてください。このような情報は、必要に応じてバックグラウンドで読み込みを続行し、使用可能になったら表示してください。

**中断後のオーディオ再生の再開は適切な場合にのみ行う。** 例えば、電話着信のような一時的な中断のあとは、オーディオを再開するのが適切です。Siriによって開始されたミュージックプレイリストなど、永続的な中断の場合、再開は適切ではありません。再開が適切であるような中断が始まったときにオーディオが再生されていた場合は、中断が終わったら再生を再開してください。

**必要な場合は音量を自動調整するが、全体の音量は変更しない。** アプリでは、適切にオーディオをミックスできるように音量レベルを独立して相対的に調整できますが、最終出力音量はユーザが制御する必要があります。

## [レイアウト](/jp/design/human-interface-guidelines/carplay#Layout)

CarPlayは、さまざまなピクセル密度とアスペクト比のディスプレイ解像度に幅広く対応しています。アプリのアイコンやインターフェイスはディスプレイの解像度に応じて自動的にスケーリングされるため、常にほぼ同じサイズで画面に表示されます。下の表に、一般的な画面サイズを示します。

サイズ（ピクセル）| アスペクト比  
---|---  
800x480| 5:3  
960x540| 16:9  
1280x720| 16:9  
1920x720| 8:3  
  
**高価値の有益な情報を見やすく表示して、運転席から簡単に読み取れるようにする。** 不必要な細かい情報や視覚的な装飾で画面を雑然とさせないでください。

**アプリ全体で一貫した見た目を維持する。** 基本的に、同じような機能を持つ要素は同じような見た目になるようにしてください。

**主要なコンテンツを目立たせて操作しやすくする。** 大きい項目は小さい項目より重要なものに見え、タップもしやすくなります。一般的に、最も重要なコンテンツやコントロールを画面の上半分に配置します。

## [カラー](/jp/design/human-interface-guidelines/carplay#Color)

カラーはインタラクティブな要素を表現し、活気を与え、視覚的な連続性をもたらします。

**通常はアプリのロゴと調和した、色数を抑えたカラーパレットを優先して使用する。** カラーを控えめに使うことで、ブランドのイメージを効果的に伝えることができます。

**インタラクティブな要素とインタラクティブでない要素に同じカラーを使わない。** インタラクティブな要素とインタラクティブでない要素に同じカラーを使うと、どこをタップしてよいか分かりにくくなります。

**実際の車のさまざまな照明条件でアプリのカラースキームをテストする。** 照明は、時刻、天気、ガラス窓の色合いなどによって大きく変わります。設計時にコンピュータに表示されたカラーが現実世界でアプリを使用したときと常に同じに見えるとは限りません。色の明るさが夜間の運転に及ぼす影響や、コントラストの低いカラーが直射日光の下で不鮮明になることを考慮に入れてください。必要な場合は、大半のユースケースで最適な視聴体験を実現できるようにカラーを調整してください。

**ダークモードとライトモードのどちらでもアプリが見栄え良く表示されるようにする。** CarPlayはライトとダークの両方の見た目に対応しており、照明条件に応じて現在の見た目が自動的に調整されます。

**誰にとっても内容が効果的に伝わるカラーを選択する。** カラーの見え方や解釈はユーザによって異なります。ユーザ体験の向上に役立つカラーの使い方についてのガイダンスは、[インクルーシブなカラー](https://developer.apple.com/jp/design/human-interface-guidelines/color#Inclusive-color)を参照してください。

## [アイコンと画像](/jp/design/human-interface-guidelines/carplay#Icons-and-images)

CarPlayは、横向きと縦向きの両方の表示に対応し、@2x（低解像度）と@3x（高解像度）の両方の倍率に対応しています。

**アプリ内のすべてのCarPlayアートワークについて、倍率@2xおよび@3xの高解像度画像を提供する。** 車載ディスプレイの解像度とサイズに応じて、システムが自動的に適切な画像を表示し、適宜スケーリングします。

**iPhoneのアプリアイコンを忠実に反映する。** 優れたデザインのアプリアイコンはCarPlayでもiPhoneでも効果的に使用できるため、別のデザインを用意する必要はありません。

**アイコンの背景を黒にしない。** アイコンがディスプレイの背景と一体化しないように、黒い背景を明るくするか、枠線を追加します。

CarPlayアプリのアイコンは以下のサイズで作成します。

@2x（ピクセル）| @3x（ピクセル）  
---|---  
120x120| 180x180  
  
## [エラー処理](/jp/design/human-interface-guidelines/carplay#Error-handling)

CarPlayアプリでは、エラーを適切に処理し、どうしても必要な場合にのみユーザに報告するようにしてください。

**エラーの報告は接続されたiPhoneではなくCarPlayで行う。** ユーザに問題を知らせる必要がある場合は、CarPlayで分かりやすく表示しましょう。エラーを見たり解決したりするために、iPhoneを手に取るようユーザに指示しないでください。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/carplay#Platform-considerations)

 _iOSに追加の考慮事項はありません。iPadOS、macOS、tvOS、visionOS、watchOSには対応していません。_
