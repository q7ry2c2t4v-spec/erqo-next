# ページコントロール

ページコントロールでは、インジケータの画像が1行に並んで表示されます。各インジケータがフラットリスト内の1ページに対応します。

![図案化されたページコントロール。アクティブなページを示すインジケータがあります。画像は全体が、6色のオリジナルAppleロゴを連想させる赤色になっています。](https://docs-assets.developer.apple.com/published/e97626512d6b5f210d4485847815e30f/components-page-dots-intro%402x.png)

スクロールするインジケータの行は、リスト内を移動しながら目的のページを見つけるのに役立ちます。ページコントロールは任意の数のページを扱えるので、ユーザがカスタムのリストを作成できる場合に特に便利です。

デフォルトでは、ページコントロールは利用できるページを一連の小さな点のインジケータで表します。塗りつぶされた点が現在のページを表します。点は常に等間隔に配置され、多すぎてウインドウに収まりきらない場合は表示が切れます。

## [ベストプラクティス](/jp/design/human-interface-guidelines/page-controls#Best-practices)

**ページコントロールは、順序付きのページリスト内の移動を表す目的で使う。** ページコントロールでは、階層型のページや不連続なページの関係性は表せません。複雑なナビゲーションが必要な状況では、サイドバーや分割ビューの使用を検討してください。

**ページコントロールはビューまたはウインドウの下部中央に配置する。** ユーザがページコントロールをすぐに見つけられるよう、常にビューの下部近くに中央揃えで横向きに配置します。

**ページコントロールは任意の数のページを扱えるが、あまり多くのインジケータは表示しない。** 点が10個を超えるあたりから、素早く数えるのが難しくなります。アプリで11個以上のページインジケータを表示したい場合は、別の配置方法を検討してください。例えばグリッドに配置すると、ユーザは順番に縛られずにページ間を移動できます。

## [インジケータのカスタマイズ](/jp/design/human-interface-guidelines/page-controls#Customizing-indicators)

デフォルトのページコントロールでは、システムが提供する点の画像がすべてのインジケータに使用されます。独自の画像を表示して、特定のページを見分けやすくすることもできます。例えば「天気」では、「location.fill」シンボルを使って現在地のページを区別しています。

アプリやゲームの改善につながる場合は、すべてのインジケータのデフォルト画像として使用するカスタム画像を用意できます。特定のページに別の画像を指定することもできます。デベロッパ向けのガイダンスは、[`preferredIndicatorImage`](/documentation/UIKit/UIPageControl/preferredIndicatorImage)および[`setIndicatorImage(_:forPage:)`](/documentation/UIKit/UIPageControl/setIndicatorImage\(_:forPage:\))を参照してください。

**カスタムのインジケータ画像はシンプルで明確なものにする。** 複雑な図形は避け、ネガティブスペース、テキスト、内側の線は含めません。このような細かい表現を使用すると、アイコンが見づらくなり、サイズが小さくなったときに判別できなくなる可能性があります。インジケータとしてシンプルな[SF Symbols](/jp/design/human-interface-guidelines/sf-symbols)を使用するか、独自のアイコンをデザインすることを検討してください。ガイダンスは、[アイコン](/jp/design/human-interface-guidelines/icons)を参照してください。

**デフォルトのインジケータ画像は、ページコントロール全体の意味が分かりやすくなる場合のみカスタマイズする。** 例えば、リストのすべてのページにブックマークが含まれている場合は、デフォルトのインジケータ画像として「bookmark.fill」シンボルを使用できます。

**1つのページコントロールで3個以上の異なるインジケータ画像を使用しない。** 「天気」の現在地ページのように、リストに特別な意味を持つページが1つ含まれている場合は、独自のインジケータ画像を割り当ててそのページを目立たせることができます。一方、ページコントロールで複数の重要なページを示すためにいくつかの独自画像を使用すると、それぞれの画像の意味を覚えなければならないので、使いづらくなります。ページコントロールに3個以上のタイプのインジケータ画像を表示すると、それぞれの画像が明確であっても、乱雑で無秩序に見えます。

![天気アプリ。画面下端のページコントロールが強調表示されています。ページコントロールには、弱い太陽、雲、雲と太陽、雲と雨などのアイコンの組み合わせが表示されています。](https://docs-assets.developer.apple.com/published/671a959d0a2a6baf234fcc2255b9abdb/page-indicator-customization-incorrect%402x.png)

![丸に囲まれたX印が不適切な例であることを示しています。](https://docs-assets.developer.apple.com/published/209f6f0fc8ad99d9bf59e12d82d06584/crossout%402x.png)複数の種類のインジケータを使用すると、ページコントロールが煩雑に見え、使いづらくなります。

![天気アプリ。画面下端のページコントロールが強調表示されています。ページコントロールには、先頭側に現在地のシンボルが表示され、そのあとに複数の点が続いています。](https://docs-assets.developer.apple.com/published/b797a18ec978a588e29113b3a9f522b7/page-indicator-customization-correct%402x.png)

![丸に囲まれたチェックマークが適切な例であることを示しています。](https://docs-assets.developer.apple.com/published/88662da92338267bb64cd2275c84e484/checkmark%402x.png)2種類のインジケータのみを使用しているので、整然とした一貫性のある見た目になっています。

**インジケータ画像に色を付けない。** カスタムの色を使用するとコントラストが低下してしまい、現在のページのインジケータが識別しにくくなったり、ページコントロール自体が見づらくなったりするおそれがあります。インジケータの色の調整はシステムにお任せください。状況に合わせてページコントロールを使いやすく見栄えのよいものにするために、自動的に調整を行います。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/page-controls#Platform-considerations)

 _macOSには対応していません。_

### [iOS/iPadOS](/jp/design/human-interface-guidelines/page-controls#iOS-iPadOS)

ページコントロールでは、リストについての追加情報を提供するため、インジケータの見た目が調整されます。例えば、ユーザが現在のページのリスト内での相対的な位置を把握できるように、そのページのインジケータが強調表示されます。全部のインジケータがスペースに収まりきらない場合、ほかにもページがあることを示すため、両端付近のインジケータのサイズが小さくなります。

![ページコントロールの図。ページコントロールに合計9個の点が表示されています。中央の5個の点はデフォルトのサイズ、2つ目と8つ目の点はデフォルトのサイズの約半分、1つ目と9つ目の点はデフォルトのサイズの約4分の1になっています。中央の点が塗りつぶされており、リストでの現在のページの位置を示しています。](https://docs-assets.developer.apple.com/published/6413d2970c5a12e6374d83bed419293c/page-controls-many-indicators%402x.png)

ページコントロールは、タップまたはスクラブで操作します（ _スクラブ_ とは、コントロールをタッチして左右にドラッグすることです）。現在のページインジケータよりも先頭側または末尾側をタップすると、前または次のページが表示されます。iPadOSでは、ポインタを使用して特定のインジケータを操作することもできます。スクラブするとページが連続して順番に開きます。コントロールの先頭または末尾を超えてスクラブすると、素早く先頭または最後のページを開くことができます。

デベロッパ向けメモ

APIでは、 _タップ_ は _独立した操作_ ですが、 _スクラブ_ は _連続した操作_ になります。デベロッパ向けのガイダンスは、[`UIPageControl.InteractionState`](/documentation/UIKit/UIPageControl/InteractionState-swift.enum)を参照してください。

**スクラブ中はページ遷移のアニメーションを使用しない。** スクラブはかなり速く行われる可能性があります。すべての画面遷移でスクロールのアニメーションを使用すると、アプリが遅くなったり、目障りな画面の点滅が生じたりしかねません。画面遷移でスクロールのアニメーションを使用するのは、タップした場合のみにしてください。

ページコントロールに半透明の角丸四角形の背景を追加して、インジケータのコントラストを高められます。以下のいずれかのバックグラウンドスタイルから選択してください:

  * 自動 — コントロールを操作している間だけ背景が表示されます。UIの主要なナビゲーション要素がページコントロールではない場合は、このスタイルを使用してください。

  * はっきり — 常に背景が表示されます。コントロールが画面の主要なナビゲーションコントロールである場合にのみ、このスタイルを使用してください。

  * 最小限 — 背景が表示されません。リストで現在のページの位置を示したいだけで、スクラブ時の視覚フィードバックが必要ない場合は、このスタイルを使用してください。

デベロッパ向けのガイダンスは、[`backgroundStyle`](/documentation/UIKit/UIPageControl/backgroundStyle-swift.property)を参照してください。

**バックグラウンドスタイルを「最小限」にする場合はスクラバーに対応しない。** 「最小限」のスタイルでは、スクラブ時の視覚的なフィードバックが返されません。アプリでページのリストをスクラブできるようにしたい場合は、バックグラウンドスタイルを「自動」または「はっきり」にしてください。

### [tvOS](/jp/design/human-interface-guidelines/page-controls#tvOS)

**フルスクリーンページのコレクションではページコントロールを使用する。** ページコントロールは、ページ階層内に魅力的なコンテンツのページが複数並んでいる状況でフルスクリーン環境を操作するユーザを想定した設計になっています。ほかのコントロールを含めると、ページ間を移動する操作にユーザが集中しにくくなります。

### [visionOS](/jp/design/human-interface-guidelines/page-controls#visionOS)

visionOSでは、ページコントロールで、利用可能なページ数および現在のページを表せますが、ユーザがページコントロールを操作することはできません。

### [watchOS](/jp/design/human-interface-guidelines/page-controls#watchOS)

watchOSでは、横方向のページネーションのときは画面下部に、縦方向の[タブビュー](https://developer.apple.com/design/human-interface-guidelines/components/layout-and-organization/tab-views)を表示するときはDigital Crownの横にページコントロールが表示されます。縦のタブビューの使用時には、ページインジケータによって、現在のページ内のどこにいるか、またページセット内のどこにいるかが示されます。ページコントロールは、ページのコンテンツをスクロールする状態と、ほかのページにスクロールする状態とが切り替わります。

![縦方向のタブビューを含むApple Watchの画面の図。Digital Crownの横にあるページコントロールが、現在4つ目のタブを選択中であることを示しています。](https://docs-assets.developer.apple.com/published/d8ac19e2578c57035e74fe697a15d573/page-controls-watch-vertical%402x.png)

縦方向のページコントロール

![横方向のタブビューを含むApple Watchの画面の図。下部にあるページコントロールが、現在2つ目のタブを選択中であることを示しています。](https://docs-assets.developer.apple.com/published/0f915ec7b0e8755a2d60845617746f71/page-controls-watch-horizontal%402x.png)

横方向のページコントロール

**複数のビューを目的ごとに個々のページに分けるには縦方向のページネーションを使用する。** 各ページに明確な目的を与え、Digital Crownでページ間をスクロールできるようにします。watchOSの場合は、横方向のページネーションを実装したりナビゲーションをいくつもの階層に分けたりするよりも、このデザインの方が効果的です。

**各ページのコンテンツを画面の高さに収まる量に限定することを検討する。** この制約を設けることで、個々のページの目的が分かりやすく明確なものになり、より一層情報を読み取りやすいデザインに仕上がります。高さが可変のページはよく考えて使用し、可能であれば、アプリのデザインの中で、高さが固定のページのあとにのみ配置してください。
