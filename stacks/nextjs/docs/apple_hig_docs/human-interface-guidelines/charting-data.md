# データのグラフ化

データをグラフで表示すると、分かりやすく説得力のある情報伝達が可能になります。

![棒グラフのスケッチ。データ表現を示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/bc4a2c4e929441fc51874ce49b4a5129/patterns-charting-data-intro%402x.png)

グラフは複雑な情報を効率よく伝達する方法で、大量のテキストを読んで理解する手間が省けます。美しく表示されたグラフは、アプリの魅力を⼀層効果的に伝え、インターフェイスの視覚的な訴求力を高める方法でもあります。グラフの作成に使用できるコンポーネントについて詳しくは、[グラフ](/jp/design/human-interface-guidelines/charts)を参照してください。

グラフには、情報を分かりやすく伝達するための単純なグラフィックから、アプリの中核をなし、多様な視点からデータを探索する機能を備えた高度なインタラクティブ環境まで、さまざまな種類があります。単純か複雑かを問わず、グラフはユーザがデータに基づいたタスクを実行するのに役立ちます。例えば、以下のような重要なタスクがあります:

  * 履歴や予測値に基づいて傾向を分析する

  * 刻々と変化するプロセス、システム、数量の現在の状態を可視化する

  * 複数のカテゴリにわたるデータを比較して、さまざまな項目（またはさまざまな時点での同じ項目）を評価する

収集したデータをすべてグラフ化する必要はありません。目的が単にデータを提供するだけで、データの傾向を示したり、ユーザによる分析を支援したりする必要がないのであれば、スクロール、検索、並べ替えができるリストや表など、ほかの方法でデータを提供することもできます。

## [ベストプラクティス](/jp/design/human-interface-guidelines/charting-data#Best-practices)

**データセットについての重要な情報を強調したい場合にグラフを使用する。** グラフは視覚に訴えることができるため、ユーザの注意を惹きつけます。対象となるデータからユーザが読み取れる情報を明確に伝えるために、この高い視認性を利用してください。

**グラフをシンプルにし、詳細な情報は必要に応じてユーザが確認できるようにする。** グラフにできるだけ多くのデータを詰め込みたいという気持ちを抑えてください。データが多すぎると、見ただけで圧倒される使いにくいグラフになり、伝えたいデータの関係性やほかの情報が不明瞭になりかねません。提示すべきデータが大量にある、またはグラフで提供する機能がたくさんある場合は、それをユーザが自ら1つずつ有効にできる方法を検討してください。例えば、データを表示する際の詳細度や対象となるデータのサブセットを、関心の度合いに応じてユーザが選択できるように設計してもよいでしょう。また、ユーザがインタラクティブグラフの使用方法を習得する助けとして、機能の複雑度が異なる、複数バージョンのグラフを提供するのも一案です。

**アプリのすべてのグラフをアクセシブルにする。** グラフは、データを美しく、見て分かるように表示するための視覚ツールです。見て分かるように表示するだけでなく、グラフの値とコンポーネントを示すアクセシビリティラベルと、ユーザがグラフを操作できるように支援するアクセシビリティ要素の両方を提供することはきわめて重要です。ガイダンスは、[グラフのアクセシビリティの向上](/jp/design/human-interface-guidelines/charts#Enhancing-the-accessibility-of-a-chart)を参照してください。

## [効果的なグラフのデザイン](/jp/design/human-interface-guidelines/charting-data#Designing-effective-charts)

**基本的に、一般的な種類のグラフを使用する。** たいていのユーザは、棒グラフや折れ線グラフなどの一般的な種類のグラフに慣れています。このような種類のグラフを採用する方が、ユーザがグラフの読み取りかたをすでに分かっている可能性が高いはずです。ガイダンスは[グラフ](/jp/design/human-interface-guidelines/charts)を参照してください。

**斬新な方法でデータを提示するグラフを作成する必要がある場合は、ユーザがグラフを理解できるようサポートする。** 例えば、Apple WatchとiPhoneをペアリングすると、アクティビティリングが「アクティビティ」に導入されます。これは、ムーブ、エクササイズ、スタンドの達成状況に合わせてそれぞれのリングが伸びていくグラフです。

**いろいろなレベルや視点からデータを検証して見つけ出した詳細な情報を提示することで、グラフの機能を強化する。** 例えば、マクロレベルの視点からデータを見ると、合計や平均値などユーザの関心を惹きつけやすい概要情報を特定できるかもしれません。中間レベルの視点からは、どのデータがユーザにとって有用であるかを見極めるのに役立つ方法を見出せるかもしれません。一方、個々のデータポイントを調べると、ユーザの注意を特定の値や項目に惹きつける方法が分かる場合があります。多様な視点に基づくグラフを簡単に表示できる機能がアプリに備わっていると、ユーザのより積極的な関与が期待できます。

![iPhoneの株価アプリ。折れ線グラフで銘柄のパフォーマンスが示されています。表示期間は1か月です。](https://docs-assets.developer.apple.com/published/89ef1b7eb807fc4d0af73ac441678425/charts-stocks%402x.png)

![iPhoneのヘルスケアアプリの「アクティビティ」画面。3つのアクティビティリングからの情報が3つの棒グラフで表示されています。表示期間は1日です。](https://docs-assets.developer.apple.com/published/61a1813ed47092026eb016b3b550e6fe/charts-activity%402x.png)

**グラフに説明となる文字列を追加して分かりやすくする。** 分かりやすいタイトル、サブタイトル、注釈を付加すると、グラフ内の最も重要な情報が効果的に強調され、有効な指針を示すことができます。ユーザが不可欠な情報を一目で把握できるように、グラフの見出しや概要の役割を果たす簡潔な説明文を追加することもできます。例えば、天気アプリでは、24時間先までの1時間ごとの予報リストの上に「次の1時間、小雨になる可能性があるでしょう」のような、ユーザが今すぐ必要とする情報を要約したテキストが表示されます。説明となる見出しや概要があるとグラフが利用しやすくなりますが、これらはアクセシビリティラベルの代わりにはなりません。

**グラフのサイズをグラフの機能/トピック/詳細度に合わせる。** 一般にグラフは、必要な詳細情報を快適に表示できる大きさにする必要があります。また、対応したいインタラクティブ機能に必要なだけの領域を確保する必要もあります。例えば、グラフの詳細なデータ、ラベルや注釈などの文字列を読みやすくしたいと考えるのは当然ですが、グラフの対象範囲を変更したり、異なる視点からデータを調べたりするための領域も確保したい場合があるでしょう。一方、個々の項目についての情報を一目で分かるように呈示したり、より大きなグラフにおけるある時点での情報やプレビューを別のビューで表示したりするために、小さなグラフを使用する場合もあります。

**複数のグラフ間でできるだけ一貫性を維持し、例外は差異を強調する必要がある場合のみにする。** アプリ内の複数のグラフに類似した目的がある場合、通常はわざわざ別の種類やスタイルのグラフを使用して、それらのグラフ間に関係がないかのような印象を与えることは避けましょう。 さらに、アプリ内のグラフに一貫した視覚的アプローチを適用すれば、ユーザは、1つのグラフで習得したノウハウを別のグラフを理解するために活用できます。グラフ間の重要な差異を強調するのに必要な場合に、異なる種類やスタイルの使用を検討してください。

**同じデータを使用する複数のグラフ間で一貫性を維持する。** ユーザが1つのデータセットをさまざまな視点から探索できるように複数のグラフを用意する場合、グラフの種類を1つに限定し、一貫した色、注釈、レイアウト、説明となるテキストを使用して、同じデータセットを一貫して対象にしているという視覚効果を保つのは大切です。例えば、「ヘルスケアのトレンド」画面では、それぞれが固有の表示スタイルを持つ小さいグラフが複数表示され、歩数や安静時心拍数などの領域で最近の傾向が分かります。ある領域ですべてのデータを表示するグラフを選択すると、拡張バージョンのグラフでも同じスタイル、色、マーク、注釈が使用され、バージョン間の関係がすぐに分かります。
