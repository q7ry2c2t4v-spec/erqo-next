> Source: https://developer.apple.com/jp/design/human-interface-guidelines/focus-and-selection

# フォーカスと選択

フォーカスは、操作対象のオブジェクトを視覚的に確認するのに役立ちます。

![円形のインターフェイス要素を囲むフレームのスケッチ。オブジェクトへのフォーカスのロックを示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせる紫色になっています](https://docs-assets.developer.apple.com/published/d70e7edb8e5cc1672453d1ed7e69b871/inputs-focus-and-selection-intro%402x.png)

フォーカスは、コンポーネント単位のシンプルなナビゲーションを可能にするものです。ユーザは、リモコンやゲームコントローラ、キーボードなどの入力を使って、操作したいコンポーネントにフォーカスを移動します。

多くの場合、項目はフォーカスと同時に選択されます。ただし、新しいビューが開くなど、自動選択によってコンテキストが変わってユーザの注意がそれてしまうような場合は例外です。例えばtvOSの場合、ユーザは項目を探しながらリモコンを使って項目間でフォーカスを移動しますが、項目の選択は「項目を開く」または「項目のアクティベート」を意味するため、フォーカスとは別のジェスチャを実行しないと選択は行われません。

ユーザへのフォーカスの伝え方はプラットフォームごとに異なります。例えば、iPadOSとmacOSの場合は、項目の周囲にリングを描画するか項目をハイライトすることによってフォーカスを表現します。tvOSは、基本的に[視差エフェクト](/jp/design/human-interface-guidelines/images#Parallax-effect)を使って、フォーカス中の項目に奥行きと動きを与えます。このフォーカスエフェクトと操作の組み合わせは、 _フォーカスシステム_ や _フォーカスモデル_ と呼ぶこともあります。

## [ベストプラクティス](/jp/design/human-interface-guidelines/focus-and-selection#Best-practices)

**できる限りシステムが提供するフォーカスエフェクトを使う。** システム定義のフォーカスエフェクトは、Appleデバイスでの操作を補い、軽快で、滑らかで、生き生きとした体験を実現するために緻密なチューニングが施されています。これを取り入れれば、アプリに一貫性を持たせ、アプリの動作を予測しやすくし、ユーザの理解を補助することができます。カスタムのフォーカスエフェクトを作成するのは、どうしても必要な場合だけにしてください。

**ユーザの操作なしにフォーカスを変えることはしない。** フォーカスシステムは、ユーザがアプリ内の現在の場所を知る手がかりです。操作していないのにフォーカスが変わってしまうと、ユーザはフォーカスが移動した先の項目をわざわざ探さなければならず、タスクの実行に手間取ることになります。ただし、ユーザが明確に特定の方向に移動するための入力デバイス（キーボード、リモコン、ゲームコントローラなど）を使ってフォーカスを移動している場合や、フォーカスのあった項目が消えた場合は例外です。この場合は、もともとフォーカスのあった項目から1ステップで行ける項目の数は限られているため、そのいずれかの項目にフォーカスを移動しても、フォーカスインジケータがユーザの見つけにくい場所に行ってしまうことはありません。ユーザがフォーカスの移動にキーボードなどの入力デバイスを使っていない場合、次に操作したいであろう項目を予測することはできません。そのため、基本的には、フォーカスのあったオブジェクトが消えた場合にはフォーカスインジケータを非表示にするのが最善です。

**アプリ内項目へのフォーカスの移動を補助するときにはプラットフォーム内で一貫性を持たせる。** 例えば、iPadOSとmacOSの場合は、フルキーボードアクセスモードを使ってすべてのコントロールにキーボードでフォーカスを移動することができるため、項目やテキストフィールド、検索フィールドなどのコンテンツの要素をフォーカスに対応させるだけで済みます。ボタン、スライダ、トグルなどのコントロールをフォーカスに対応させる必要はありません。反対にtvOSの場合は、画面上のどの要素に移動する場合でもリモコンまたはゲームコントローラの方向ジェスチャ（または接続されたキーボードの矢印キーの押下）しか使えないため、必ずアプリ内のすべての要素をフォーカス可能にする必要があります。

**フォーカスを示す視覚的なエフェクトはプラットフォーム内で一貫性を持たせる。** 項目のリストを含むウインドウを例に考えてみます。iPadOSとmacOSは、白いテキストと、アプリのアクセントカラーと同じ色のバックグラウンドハイライトを使ってフォーカスのあるリスト項目を描画し、フォーカスのない項目は、標準のテキストカラーとグレイのバックグラウンドハイライトを使って描画します（デベロッパ向けのガイダンスは、[`UICollectionView`](/documentation/UIKit/UICollectionView)および[`NSTableView`](/documentation/AppKit/NSTableView)を参照してください）。

**基本的に、テキストフィールドや検索フィールドはフォーカスリングで囲み、リストやコレクション内の項目は強調表示する。** セルいっぱいに表示される写真などの項目に注意を引く場合はフォーカスリングを使ってもかまいませんが、リストやコレクションは、基本的に行全体を強調表示した方がユーザにとって見やすくなります。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/focus-and-selection#Platform-considerations)

 _iOSとwatchOSには対応していません。_

### [iPadOS](/jp/design/human-interface-guidelines/focus-and-selection#iPadOS)

iPadOS 15以降のフォーカスシステムでは、アプリ内のコレクションビューやその他のカスタムビューに加えて、テキストフィールド、テキストビュー、サイドバーもキーボードで操作できるようになっています。

iPadOSとtvOSのフォーカスシステムは似ています。どちらの場合も、項目にフォーカスインジケータを移動してから項目を選択することによってアクションを実行します（ガイダンスは[tvOS](/jp/design/human-interface-guidelines/focus-and-selection#tvOS)を参照してください）。このように基本的なシステムは同じですが、ユーザの操作感は少し異なります。tvOSは _方向性フォーカス_ を採用しているので、画面上のどのコンポーネントに移動する場合も操作方法は同じです。つまり、Siri Remoteをスワイプするか接続されたキーボードの矢印キーを使う方法しかありません。これに対しiPadOSでは、サイドバー、グリッド、リストなどのアプリ内の特定の領域を表す _フォーカスグループ_ を定義します。iPadOSはフォーカスグループを使って、以下の2つの異なるキーボード操作に対応できます。

  * Tabキーを押すとフォーカスがフォーカスグループ間を移動します。ユーザはこれで、サイドバー、グリッドなどのアプリ領域に移動できます。

  * 矢印キーを押すとtvOSと同様の方向性フォーカスの操作になりますが、同じフォーカスグループ内の項目間の移動に限られます。例えば、ユーザは矢印キーを使ってリストやサイドバーの項目間を移動できます。

どのコンポーネントにフォーカスがあるかは、ハローエフェクトという強調表示を使ってユーザに示すことができます。

 _ハロー_ フォーカスエフェクトは _フォーカスリング_ とも呼ばれ、コンポーネントの周囲に表示されるカスタマイズ可能なアウトラインのことです。ハローエフェクトはカスタムビューや、コレクションやリストセル内の透明部分を含まないコンテンツ（画像など）に適用できます。

![写真のコレクションビューの図。フォーカスのある写真に標準のハローエフェクトの輪郭線が付いています。](https://docs-assets.developer.apple.com/published/2bfe6fedc5a6a8ecf6d7e74e9492a096/focus-and-selection-halo-focus-effect%402x.png)

**必要に応じてハローフォーカスエフェクトをカスタマイズする。** デフォルトでは、項目の形状に基づいてハローの形状が推測されます。システムが提供するハローの見栄えがよくない場合は、角を丸めたりベジェ曲線で形状を定義したりして輪郭を整えられます。他のコンポーネントに隠れたり干渉したりする場合は、ハローの位置も調整できます。例えば、バッジがハローの上に表示されるようにしたり、親ビューがハローに干渉しないように調整したりする必要が生じるかもしれません。デベロッパ向けのガイダンスは、[`UIFocusHaloEffect`](/documentation/UIKit/UIFocusHaloEffect)を参照してください。

![写真のコレクションビューの図。フォーカスのある写真に角丸四角形のハローエフェクトの輪郭線が付いています。](https://docs-assets.developer.apple.com/published/1a84f872d0624355e89fa03b357ddd13/focus-and-selection-customized-halo%402x.png)

コンポーネントのテキストをアプリのアクセントカラーにする _強調表示_ もフォーカスの位置を示しますが、コンテンツが設定済みのコレクションビューのセルをユーザが選択すると自動的に強調表示になる点が、フォーカスエフェクトとは異なります（デベロッパ向けのガイダンスは、[`UICollectionViewCell`](/documentation/UIKit/UICollectionViewCell)を参照してください）。

![2つ目の項目が強調表示されているメニュー項目のリストの図。項目のタイトルとアイコンは赤のアクセントカラーです。](https://docs-assets.developer.apple.com/published/24ba62d4cb3c06e40eb1bdea3d88359c/focus-and-selection-highlighted-appearance%402x.png)

**カスタムビュー間で合理的な順序でフォーカスを移動する。** ユーザがTabキーを押すごとに、フォーカスは、読む順序に従って（先頭から末尾へ、上から下へ）フォーカスグループ間を移動します。システムが提供するビュー間ではフォーカスはユーザの予期した通りに移動しますが、カスタムビュー間をフォーカスが移動する順序は調整する必要があるかもしれません。例えば、縦に重なったカスタムビュー間を上から下に移動してから、右にある次のビューにフォーカスを移動したい場合、その重なった部分を1つのフォーカスグループとして認識する必要があります。デベロッパ向けのガイダンスは、[`focusGroupIdentifier`](/documentation/UIKit/UIFocusEnvironment/focusGroupIdentifier)を参照してください。

**項目の優先順位をその項目のフォーカスグループ内の重要度に合わせて調整する。** グループにフォーカスが移動した際に、そのグループの _最上位の項目_ に自動的にフォーカスが設定されると、ユーザは最もよく使用する項目を簡単に選択できます。項目の優先順位を上げると、その項目を最上位に設定できます。デベロッパ向けのガイダンスは、[`UIFocusGroupPriority`](/documentation/UIKit/UIFocusGroupPriority)を参照してください。

### [tvOS](/jp/design/human-interface-guidelines/focus-and-selection#tvOS)

**フルスクリーンの操作環境では、ユーザがフォーカスを移動するジェスチャではなくコンテンツを操作するジェスチャを使うようにする。** 項目がフルスクリーンで表示されるとフォーカスは表示されなくなります。当然ながらユーザは、ジェスチャで操作できるのはフォーカスの状態ではなく、コンテンツそのものだと判断します。

**ポインタを表示しないようにする。** 項目数が限られている場合は、広い画面で小さなポインタをドラッグするよりも、フォーカス移動の方が楽に操作できるはずです。隠された物を探したり、飛行機を操縦したりするようなゲームのプレイ中は、自由に動き回れる操作が適切ですが、メニューやその他のインターフェイス要素間を移動する場合はフォーカスモデルを使用しましょう。アプリでポインタを必要とする場合は、必ずポインタの視認性を高めて、統一感のある操作性を創出するようにしてください。

**さまざまなフォーカス状態のコンポーネントに対応できるインターフェイスを設計する。** tvOSでは、フォーカス可能な項目は最大で5つの状態に変化します。それぞれはっきりと異なる見た目を持ちます。フォーカスされている項目は通常拡大されるので、拡大時でもくっきりとした見た目が損なわれないような素材を用意する必要があります。拡大された項目と周囲のインターフェイスとの間に十分な間隔を確保する必要もあります。

状態| 説明  
---|---

### [visionOS](/jp/design/human-interface-guidelines/focus-and-selection#visionOS)

visionOSは、iPadOSやtvOSと同じフォーカスシステムに対応しており、キーボードやゲームコントローラなどの接続された入力デバイスを使ってアプリやシステムを操作できます。

注意

ユーザが操作したいオブジェクトを判別するために仮想オブジェクトを注視すると、フォーカスエフェクトではなく _ホバーエフェクト_ で視覚的なフィードバックが提供されます（ガイダンスは、[視線](/jp/design/human-interface-guidelines/eyes)を参照してください）。ホバーエフェクトは、フォーカスシステムとは関係がありません。
