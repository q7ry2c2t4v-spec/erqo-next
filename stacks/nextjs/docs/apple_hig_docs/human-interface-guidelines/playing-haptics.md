# 触覚フィードバックの提供

触覚フィードバックを提供してユーザの触覚に働きかけると、物質的世界で慣れ親しんだ感覚をアプリやゲームに導入することができます。

![3つの円が水平に並んでわずかに重なり合う線のスケッチ。バイブレーションを示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/bf4c1053afaa9f1b01ba1d48f3392a18/patterns-playing-haptics-intro%402x.png)

ユーザが使用しているプラットフォームやデバイスによっては、視覚や聴覚のフィードバックに加えて触覚フィードバックを提供することができます。例えば、対応しているiPhoneのモデルでは、スイッチ、スライダ、ピッカーなどのコンポーネントで自動的に触覚フィードバックが提供されます。Apple Watchでは、Taptic Engineで多数の内蔵のフィードバックパターン用に触覚フィードバックが提供され、watchOSによってそれに音が組み合わされます。感圧タッチトラックパッドを搭載したMacのアプリは、コンテンツのドラッグ中や、強めのクリックでメディアコントロールの速度を変更するときに、触覚フィードバックを提供することができます。

内蔵の触覚フィードバック機能に加え、一部の外部入力デバイスで触覚フィードバックを提供することもできます。例えば、以下のような使い方があります:

  * iPadOS、macOS、tvOS、またはvisionOSのアプリやゲームでは、[ゲームコントローラ](/jp/design/human-interface-guidelines/game-controls)で触覚フィードバックを提供できます（デベロッパ向けのガイダンスは、[ゲームコントローラでの触覚フィードバックの提供](/documentation/CoreHaptics/playing-haptics-on-game-controllers)を参照してください）。

  * [Apple Pencil Pro](/jp/design/human-interface-guidelines/apple-pencil-and-scribble)と一部のトラックパッドでは、特定のiPadのモデルに接続されているときに触覚フィードバックを提供できます。（Apple Pencilの機能と互換性について詳しくは、[Apple Pencil](https://www.apple.com/apple-pencil/)を参照してください。）

## [ベストプラクティス](/jp/design/human-interface-guidelines/playing-haptics#Best-practices)

**ドキュメントに記載されている用途に従ってシステムが提供する触覚パターンを使用する。** 標準の触覚フィードバックは、標準コントロールを操作するときに常に提供されるため、ユーザはすでに認識しています。ドキュメントに記載されている使用パターンが、アプリやゲームにとって適切でない場合は、そのパターンに別の意味を持たせて使用するのは避けてください。代わりに、汎用的なパターンを使用するか、可能な場合は、独自のパターンを作成してください。ガイダンスは、[カスタムの触覚フィードバック](/jp/design/human-interface-guidelines/playing-haptics#Custom-haptics)を参照してください。

**アプリまたはゲーム全体で一貫性のある触覚フィードバックを使用する。** ユーザが特定の触覚パターンと特定の体験を結びつけることができるように、触覚フィードバックとそれを発生させるアクションには、明確な因果関係を持たせることが重要です。因果関係が明白でない触覚フィードバックは、ユーザを混乱させ、無駄なものと見なされる可能性があります。例えば、ゲームのキャラクタがミッションを達成できなかったときに特定の触覚パターンを提供する場合、ユーザはそのパターンから否定的な結果を連想します。あるレベルをクリアしたときなど、肯定的な結果に同じ触覚パターンを使用すると、ユーザは混乱してしまいます。

**触覚フィードバックはアプリまたはゲームでほかのフィードバックを補完する形で使用する。** 物質世界で通常そうであるように、視覚、聴覚、触覚によるフィードバックが協調して動作することで、ユーザ体験は一貫性のある自然なものになります。例えば、通常は触覚フィードバックの強さや鮮明さを、一緒に表示するアニメーションの強さや鮮明さと合わせるようにしてください。サウンドと触覚フィードバックを同期させることもできます。デベロッパ向けのガイダンスは、[触覚フィードバックで豊かなアプリ体験を提供する](/documentation/CoreHaptics/delivering-rich-app-experiences-with-haptics)を参照してください。

**触覚フィードバックは多用しすぎないようにする。** 触覚フィードバックは、たまに提供されると効果的ですが、頻繁に使うとしつこく感じられることがあります。ユーザテストを行うと、ユーザが最も心地よく感じるバランスを見つけるうえで役立ちます。多くの場合は、ユーザが意識しないくらいでありながら、無効にすると物足りなく感じられる程度が最適な体験です。

**ほとんどのアプリでは個々のイベントを補う程度の短い触覚フィードバックを提供する。** ゲームプレイフローに伴って生じる長く続く触覚フィードバックで操作性を高めることはできますが、アプリでの長く続く触覚フィードバックは、フィードバックの意味を薄くさせ、タスクからユーザの気を逸らしてしまう可能性があります。Apple Pencil Proでは、例えば、連続する触覚フィードバックや長く続く触覚フィードバックで手書きや描画の操作が分かりやすくなる傾向はなく、Apple Pencilを持っているのが不快になる可能性すらあります。

**触覚フィードバックはオプションにする。** 触覚フィードバックはオフにできるようにし、触覚フィードバックがなくてもアプリを使えるようにします。

**触覚フィードバックの提供によって、別のユーザ体験が影響を受ける可能性があることを認識する。** 設計上、触覚フィードバックを行うと、ユーザに振動を感じさせるような力が発生します。発生した振動によって、カメラ、ジャイロスコープ、マイクなどのデバイスの機能に関わる操作性が損なわれないようにしてください。

## [カスタムの触覚フィードバック](/jp/design/human-interface-guidelines/playing-haptics#Custom-haptics)

ゲームでは、カスタムの触覚フィードバックを使用してゲームのプレイ体験を向上させることが多くあります。あまり一般的ではありませんが、ゲーム以外のアプリでもカスタムの触覚フィードバックを使用して、高度で楽しい体験を実現することがあります。

ユーザの入力や状況に応じてダイナミックに変化するカスタムの触覚パターンをデザインできます。例えば、ゲームのキャラクタが木からジャンプしたときにプレイヤーが感じるフィードバックは、キャラクタがその場でジャンプしたときよりも強いものになるでしょう。また、衝突や命中などのような物理的な衝撃をともなう体験は、足音や危険の接近といった実際の衝撃を伴わない体験とは異なる形で表現されるでしょう。

カスタムの触覚パターンを生成するために使用できる基本的な構成要素は2つあります。

  *  _一時的_ なイベントは、短時間で簡潔に表現され、多くの場合、タップや短い刺激のように感じられます。一時的なイベントの例としては、ホーム画面でフラッシュライトボタンをタップしたときの体験が挙げられます。

  *  _継続的_ なイベントは、バイブレーションが持続するように感じられます。例えば、メッセージのレーザーエフェクトがこれに当たります。

カスタムの触覚フィードバックを生成するために使用する触覚イベントの種類とは別に、 _鮮明さ_ （sharpness）と _強さ_ （intensity）を操作できます。鮮明さは、触覚体験を体で感じることができる波形に変換する方法と考えることができます。鮮明さを指定することで、体験の意図をシステムに伝えることができます。例えば、鮮明さの値を使用すると、柔らさ、丸み、有機的といった感覚や、堅さ、正確さ、機械的といった感覚を伝えることができます。強さは、その名の通り、触覚フィードバックの強度を表します。

一時的なイベントと継続的なイベントを組み合わせ、鮮明さと強さを変化させ、必要に応じてオーディオコンテンツを追加することで、非常に幅広い触覚体験を作成できます。デベロッパ向けのガイダンスは、[Core Haptics](/documentation/CoreHaptics)を参照してください。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/playing-haptics#Platform-considerations)

### [iOS](/jp/design/human-interface-guidelines/playing-haptics#iOS)

対応しているiPhoneのモデルでは、以下の方法で体験に触覚フィードバックを追加できます。

  * Appleがデザインしたシステム触覚フィードバックがデフォルトで再生される標準UIコンポーネント（[トグル](https://developer.apple.com/jp/design/human-interface-guidelines/toggles)、[スライダ](https://developer.apple.com/jp/design/human-interface-guidelines/sliders)、[ピッカー](https://developer.apple.com/jp/design/human-interface-guidelines/pickers)など）を使用する。

  * 適切な場合は、フィードバックジェネレータを使用して、あらかじめ定義された触覚パターンを再生する。これらのパターンは、[通知](https://developer.apple.com/jp/design/human-interface-guidelines/playing-haptics#Notification)、[刺激](https://developer.apple.com/jp/design/human-interface-guidelines/playing-haptics#Impact)、[選択](https://developer.apple.com/jp/design/human-interface-guidelines/playing-haptics#Selection)のいずれかのカテゴリに含まれています（デベロッパ向けのガイダンスは、[`UIFeedbackGenerator`](/documentation/UIKit/UIFeedbackGenerator)を参照してください）。

### [macOS](/jp/design/human-interface-guidelines/playing-haptics#macOS)

Magic Trackpadが利用できる場合は、アプリでドラッグ操作や強めのクリックが行われた際に、次の3つの触覚パターンのいずれかで応答できます。

触覚フィードバックのパターン| 説明  
---|---  
アライメント（alignment）| ドラッグする項目の位置が合ったことを知らせます。例えば、描画アプリで図形をドラッグして別の図形と位置を揃えるときにこのパターンを使用できます。別の例として、オブジェクトを拡大/縮小して特定のサイズに合わせる場合、好みの位置にオブジェクトを配置する場合、ビデオアプリのスクラバーなどで開始点/終了点や最小値/最大値に到達した場合などにこのタイプのフィードバックを使用できます。  
レベル変更（levelChange）| 異なる加圧レベル間を移動したことを知らせます。例えば、ビデオプレーヤーで早送りボタンを押して再生速度を増減できるようにする場合、特定の強さに達したときに触覚フィードバックを提供します。  
汎用（generic）| ほかのパターンが該当しない場合の汎用的なフィードバックを提供します。  
  
デベロッパ向けのガイダンスは、[`NSHapticFeedbackPerformer`](/documentation/AppKit/NSHapticFeedbackPerformer)を参照してください。
