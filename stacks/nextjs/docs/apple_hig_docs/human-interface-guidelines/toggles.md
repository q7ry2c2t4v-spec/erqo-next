# トグル

トグルは、オンとオフのように2つの相反する状態を選択するためのコンポーネントで、選択した状態に合わせて見た目が変化します。

![図案化された2つのラベルつきスイッチコントロール。画像全体が、6色の初代Appleロゴの赤色を連想させる赤みを帯びています。](https://docs-assets.developer.apple.com/published/e27a05479d7c9b3f13492b0124f0f23f/components-toggles-intro%402x.png)

トグルには、スイッチやチェックボックスなど、さまざまなスタイルがあります。プラットフォームごとにこれらのスタイルを異なる方法で使用できます。ガイダンスは、[プラットフォームの考慮事項](/jp/design/human-interface-guidelines/toggles#Platform-considerations)を参照してください。

そのほか、状態に応じて見た目が変わるトグル風のボタンを全プラットフォームで利用できます。デベロッパ向けのガイダンスは、[`ToggleStyle`](/documentation/SwiftUI/ToggleStyle)を参照してください。

## [ベストプラクティス](/jp/design/human-interface-guidelines/toggles#Best-practices)

**コンテンツやビューの状態に影響する2つの相反する値をユーザに選択させるために使う。** トグルは常に、何かの状態を管理するために使用します。項目のリストからの選択のようなほかのタイプのアクションに対応する必要がある場合は、[ポップアップボタン](/jp/design/human-interface-guidelines/pop-up-buttons)などほかのコンポーネントを使ってください。

**トグルの影響を受ける設定、ビュー、コンテンツを明確にする。** 通常であれば、何のオン/オフを切り替えるのかは周囲のコンテキストから十分に理解できます。場合によっては（多くの場合はmacOS）、トグルによって何の状態が制御されるかを説明するラベルを付加してもよいでしょう。トグルとして動作するボタンを使う場合は、基本的に、その目的を伝えるインターフェイスアイコンを付け、その時々の状態に基づいて見た目を更新してください（通常は背景を変化させます）。

**トグルの状態の違いが明らかに分かるようにする。** 例えば、色の塗りつぶしのあり/なし、背景の図形の表示/非表示、内側の表示の変化（例: チェックマークやドットを表示する）などでトグルのオン/オフを示す方法が考えられます。色の違いだけで状態を伝える方法は避けてください。すべてのユーザが色の違いを識別できるとは限りません。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/toggles#Platform-considerations)

 _tvOS、visionOS、watchOSに追加の考慮事項はありません。_

### [iOS/iPadOS](/jp/design/human-interface-guidelines/toggles#iOS-iPadOS)

**スイッチスタイルのトグルはリストの行内でのみ使用する。** 何の状態を制御するためのスイッチかはその行の内容を見れば分かるので、この場合にラベルを付加する必要はありません。

**スイッチのデフォルトのカラーは必要な場合にのみ変更する。** デフォルトの緑色はほとんどの状況で効果的ですが、代わりにアプリのアクセントカラーを使いたい場合もあるでしょう。その場合は、カラーあり/なしの差が十分に知覚できるカラーを使用してください。

![2行のリストの行の図。1つの行にアクティブなスイッチのトグル、もう1つの行に非アクティブなスイッチのトグルがあります。アクティブなトグルは標準的なスイッチのカラーの緑色です。](https://docs-assets.developer.apple.com/published/6d29ec492c4d6e931a302512a3c5a947/toggles-ios-default-color%402x.png)標準的なスイッチのカラー

![2行のリストの行の図。1つの行にアクティブなスイッチのトグル、もう1つの行に非アクティブなスイッチのトグルがあります。アクティブなトグルはカスタムのスイッチのカラーの紫色です。](https://docs-assets.developer.apple.com/published/9b7cedce173d7b8238b10973cf1831e1/toggles-ios-custom-color%402x.png)カスタムのスイッチのカラー

**リスト形式以外ではスイッチではなくボタンにトグルの機能を持たせる。** 例えば電話アプリでは、フィルタボタンのトグルを使って、ユーザが最近の通話をフィルタリングできるようにします。 トグルがアクティブになると青いハイライトが表示され、非アクティブになるとハイライトが消えます。

![iPhoneの電話アプリの上半分のスクリーンショット。最近の不在着信のフィルタリングされたリストが表示されています。上部の末尾側にあるフィルタボタンには、トグルがアクティブであることを示す青いハイライトがあります。](https://docs-assets.developer.apple.com/published/c42a244cdbc58392e80b9bb729c5e561/toggles-ios-phone-filter-on%402x.png)

電話アプリでは、トグルを使って、最近のすべての通話とさまざまなフィルタオプションを切り替えます。ユーザがフィルタを選択すると、シンボルの背後にカスタムの背景が描画されたトグルが表示されます。

![iPhoneの電話アプリの上半分のスクリーンショット。最近のすべての通話が表示されています。上部の末尾側にあるフィルタボタンにはハイライトがなく、トグルが非アクティブであることを示しています。](https://docs-assets.developer.apple.com/published/4ebed4b3dfafed43d64c357698df4706/toggles-ios-phone-filter-off%402x.png)

ユーザがメインの「履歴」表示に戻ると、シンボルの背後に何もないトグルが表示されます。

**ボタンの目的を説明するラベルはなるべく付けない。** インターフェイスアイコンを作成し、さらに背景画像が交互に切り替わるようにすれば、ユーザはボタンの役割を理解できます。デベロッパ向けのガイダンスは、[`changesSelectionAsPrimaryAction`](/documentation/UIKit/UIButton/changesSelectionAsPrimaryAction)を参照してください。

### [macOS](/jp/design/human-interface-guidelines/toggles#macOS)

macOSは、スイッチスタイルに加えて、チェックボックススタイルのトグルに対応しています。類似の動作をするラジオボタンも使えます。

**スイッチ、チェックボックス、ラジオボタンは、ウインドウフレームではなくウインドウボディ領域で使用する。** 特に、ツールバーやステータスバー内でこれらのコンポーネントを使わないでください。

#### [スイッチ](/jp/design/human-interface-guidelines/toggles#Switches)

**強調したい設定にはスイッチを優先する。** スイッチはチェックボックスよりも視覚的な重みがあるので、チェックボックスで制御するような機能よりも大きな機能に使用した方がスマートな印象になります。例えば、個々の細かい設定ではなく、設定のグループ全体のオン/オフを切り替えるような場合にスイッチを使用することをおすすめします。デベロッパ向けのガイダンスは、[`switch`](/documentation/SwiftUI/ToggleStyle/switch)を参照してください。

**グループ化したフォームでは、ミニスイッチを使用して1列の設定を制御することを検討する。** ミニスイッチの高さは、ボタンなどのコントロールの高さと同じなので、列が一貫した高さになります。グループ化したフォームの中で、設定を階層で表示する必要がある場合は、プライマリの設定には通常のスイッチを、従属する設定にはミニスイッチを使用できます。デベロッパ向けのガイダンスは、[`GroupedFormStyle`](/documentation/SwiftUI/GroupedFormStyle)および[`ControlSize`](/documentation/SwiftUI/ControlSize)を参照してください。

**基本的にチェックボックスをスイッチに置き換えることはしない。** インターフェイスですでにチェックボックスを使っているのであれば、たいていの場合、チェックボックスのままにするのが最善です。

#### [チェックボックス](/jp/design/human-interface-guidelines/toggles#Checkboxes)

チェックボックスは小さな正方形のボタンです。オフのときは中身が空白で、オンにするとチェックマークが付きます。ボタンの状態が部分的にオンの場合にダッシュが表示されることもあります。通常、チェックボックスの末尾側に、タイトルが表示されます。編集可能なチェックリストでは、タイトルやその他のコンテンツなしでチェックボックスが表示されることもあります。

**設定の階層構造を示す必要がある場合はスイッチではなくチェックボックスを使う。** チェックボックスは、位置を揃えたり、グループ構造を伝えたりするのに便利な見た目をしています。例えば、あるチェックボックスの状態によってその従属チェックボックスの状態が決まるような場合は、その依存関係を配置（通常はチェックボックスの先頭側を揃える）とインデントで表現できます。

![チェックボックスが2階層構造になっている場合のレイアウトの図。](https://docs-assets.developer.apple.com/published/44ae5d54f0f3c08d9bae81997bc9bbf8/checkbox-alignment%402x.png)

**相互排他的なオプションを3つ以上示す必要がある場合はラジオボタンの使用を検討する。** ユーザが単なるオン/オフ以上のオプションから選ぶ必要がある場合は、ラジオボタンで各選択肢に固有のラベルを付けると、設定の意図が明確に伝わりやすくなります。

**チェックボックス同士の関係が不明瞭な場合は、チェックボックスのグループ全体を説明するラベルの使用を検討する。** オプションのグループについての説明を追加し、そのラベルのベースラインをグループの1つ目のチェックボックスに合わせます。

**チェックボックスの状態をその見た目に正確に反映する。** チェックボックスの状態は、オン、オフ、または部分的にオンのいずれかです。1つのチェックボックスを使って複数の従属チェックボックス全体のオン/オフを制御できる場合は、従属チェックボックスの状態が統一されていない状況を「部分的にオン」の状態で表します。この状況は例えば、すべてのテキストスタイルのオン/オフをまとめて切り替えることができるテキスト設定に加えて、ボールド、イタリック、アンダーラインなどのスタイル設定を個別に選択できるようになっている場合などに発生します。デベロッパ向けのガイダンスは、[`allowsMixedState`](/documentation/AppKit/NSButton/allowsMixedState)を参照してください。

![オン状態のチェックボックスを示す図。小さい角丸正方形の内部が青で塗りつぶされ、白いチェックマークが表示されています。](https://docs-assets.developer.apple.com/published/f60cc3ddbea31509d83e204d963cb1d0/checkbox-selected%402x.png)オン

![オフ状態のチェックボックスを示す図。見た目は内部が空白の小さい角丸正方形です。](https://docs-assets.developer.apple.com/published/ffd72e78175dc69a27016e8454030b71/checkbox-deselected%402x.png)オフ

![部分的にオン状態のチェックボックスを示す図。小さい角丸正方形の内部が青で塗りつぶされ、白いハイフンが表示されています。](https://docs-assets.developer.apple.com/published/67efc6dab34453404f164acef3bac84d/checkbox-mixed%402x.png)部分的にオン

#### [ラジオボタン](/jp/design/human-interface-guidelines/toggles#Radio-buttons)

ラジオボタンは小さい円形のボタンで、後ろにラベルが付きます。2つから5つのグループで表示されるのが普通で、相互排他的なオプションを提示するのに使用します。

![5つの項目が1列に並んでいる図。それぞれ、「ラジオボタンのラベル」というテキストの前にラジオボタンがあります。3つ目の項目のラジオボタンは塗りつぶされていて、選択された項目であることを示しています。](https://docs-assets.developer.apple.com/published/92a02e557fd632e80dae802590cecf97/radio-button-example%402x.png)

ラジオボタンの状態は、選択（塗りつぶされた円）または選択解除（空の円）のいずれかです。ラジオボタンで部分的にオンの状態を示す（ダッシュを表示する）こともできますが、ラジオボタンを追加すれば複数の状態を表せるので、部分的にオンの状態が役立つ状況はほとんどありません。ある設定値や項目が部分的にオンの状態であることを示す必要がある場合は、チェックボックスを使用してください。

![選択されたラジオボタンを示す図。見た目は内部が暗い色で塗りつぶされ、中央に白い点がある小さい円です。](https://docs-assets.developer.apple.com/published/91c45b3934ecd18b42b2bb72e64ca702/radio-button-selected%402x.png)選択

![選択解除されたラジオボタンを示す図。見た目は内部が空白の小さい円です。](https://docs-assets.developer.apple.com/published/1bec25f63381d81a41f885e1338eb571/radio-button-deselected%402x.png)選択解除

**相互排他的なオプションを提示する場合はラジオボタンの使用を検討する。** グループ内のオプションの複数選択を可能にする必要がある場合は、チェックボックスを使います。

**1つのグループに並べるラジオボタンの数を多くしすぎない。** ラジオボタンがずらずらと並んで場所を取っていると、ユーザに威圧感を与えかねません。オプションは5つくらいを目安にし、それより多くなる場合は[ポップアップボタン](/jp/design/human-interface-guidelines/pop-up-buttons)などのコンポーネントの使用を検討してください。

**オン/オフを選べる設定を1つだけ提示する場合はチェックボックスの使用を検討する。** オン/オフの切り替えは1つのラジオボタンでもできますが、チェックボックスのチェックマークの有無で現在の状態を表した方が一目で状態を把握できます。2つの相反する状態を1つのチェックボックスでは明確に表現できない例外的なケースでは、ラジオボタンを2つ一組で使って、それぞれの状態を明確に示すラベルを付けます。

**ラジオボタンを横に並べる場合は間隔を一定にする。** 最も長いボタンラベルを表示するのに必要なスペースを計算し、間隔をその値で統一してください。

![3つの項目が1行に並んでいる図。各項目の前に、ラジオボタンがあります。1つ目と3つ目の項目には長いテキストラベルがあり、2つ目の項目には短いラベルがあります。各項目が占めている横方向のスペースは同じです。2つ目の項目の前に塗りつぶされたラジオボタンがあり、選択された項目であることを示しています。](https://docs-assets.developer.apple.com/published/9d08d95820eef21f557e8333858dda5d/radio-button-equal-spacing%402x.png)
