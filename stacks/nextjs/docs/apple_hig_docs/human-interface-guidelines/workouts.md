# ワークアウト

優れたワークアウト/フィットネス体験は、ユーザが現在行っているアクティビティに集中し、デバイスで進捗を記録することを促します。

![走っている人のスケッチ。エクササイズを示しています。画像では長方形と円形のグリッド線が重なっており、6色のオリジナルAppleロゴをイメージさせるオレンジ色になっています。](https://docs-assets.developer.apple.com/published/265a8755bf7d81662149919447f27224/patterns-workouts-intro%402x.png)

Apple Watchは、さまざまなワークアウトの実行中に装着することができ、iPhoneやiPadは、ウォーキング、車椅子のプッシュ、ランニングのようなフィットネスアクティビティで携帯されることがあります。一方、iPad Pro、Mac、Apple TVのような大型または据え置き型のデバイスは、ユーザが1人またはグループでライブ/収録済みのワークアウトセッションに参加するために使用される傾向があります。

Apple Watch、iPhone、iPadでユーザの目標達成をサポートするワークアウト体験を作成するには、デバイスのアクティビティデータを活用し、馴染みのあるコンポーネントを使用してフィットネスの指標を表示します。

## [ベストプラクティス](/jp/design/human-interface-guidelines/workouts#Best-practices)

**watchOSのフィットネスアプリでは、ワークアウトセッションを使用して有益なデータと適切なコントロールを提供する。** フィットネスアプリのワークアウトセッションでは、手首を上げていない間もwatchOSによってアプリが表示され続けるため、ユーザにとって最も気になるワークアウトデータを提供することが重要です。例えば、経過時間や残り時間、カロリー消費、移動距離などを表示して、ラップマーカーやインターバルマーカーのような関連するコントロールを提供できます。

**関連性のない情報でユーザのワークアウトを妨げないようにする。** 例えば、ワークアウト中に、提案されたワークアウトのリストを確認したり、アプリのほかの部分にアクセスしたりする必要はありません。「ワークアウト」を含むwatchOSの多くのワークアウトアプリで使用される配置は次の通りです:

![「ワークアウト」の「屋外ウォーキング」ワークアウトで表示される左端画面のスクリーンショット。左上隅から時計回りで、「終了」、「再開」、「新規」、「セグメント」の各ボタンがあります。](https://docs-assets.developer.apple.com/published/07a63f4614d3880d7ec835a079dccdb8/workouts-large-buttons%402x.png)

一番左の画面には、進行中のセッションを制御する大きなボタン（「終了」、「再開」、「新規」など）が表示されます。

![「ワークアウト」の「屋外ウォーキング」ワークアウトで表示される中央画面のスクリーンショット。5行のデータが表示されています。画面には、上から順に経過時間、アクティブカロリー、現在の心拍数、平均ペース、高度が表示されています。](https://docs-assets.developer.apple.com/published/d25b9d9b15ddfa287e08e855a0d5e498/workouts-metrics%402x.png)

指標とその他のデータが専用の画面に表示され、一目で確認できます。

![一番右の「ワークアウト」画面のスクリーンショット。現在再生中のミュージックに関する情報が表示されています。](https://docs-assets.developer.apple.com/published/1193b2a0c79dcc38858f4557f40fa519/workouts-media-playback%402x.png)

対応している場合は、メディアの再生コントロールが一番右の画面に表示されます。

**アクティブなワークアウトをはっきりと区別できる外観にする。** ワークアウト中には、アクティブなセッションを一目で確認できることが望まれます。指標ページは、値がリアルタイムでアップデートされるため、セッションがアクティブであることを示すのに適しています。アップデートされる値を表示するだけでなく、独自のレイアウトを使用して指標画面をさらに目立たせることもできます。

**簡単に見つけてタップできるワークアウトコントロールを提供する。** ユーザがワークアウトを簡単に一時停止、再開、停止できるようにするだけでなく、セッションが開始または停止したことを示す明確なフィードバックを提供してください。

**ワークアウト中にセンサーデータを使用できない場合、どのようなヘルスケア情報がアプリで記録されるのか分かるようにする。** 例えば、水中では心拍数が計測されない場合がありますが、アプリではユーザが泳いだ距離や消費したカロリー量などのデータを記録できます。アプリでワークアウトの種類として _水泳_ や _その他_ に対応する場合は、システム提供のワークアウトアプリで使用されるのと同様の言い回しで状況を説明してください（下記参照）。

| ワークアウトアプリのテキスト例  
---|---  
![丸に囲まれたチェックマークは正しい使い方を示しています。](https://docs-assets.developer.apple.com/published/88662da92338267bb64cd2275c84e484/checkmark%402x.png)| プールスイミング中はGPSは使用されません。また、水中では心拍数が計測されない場合がありますが、カロリー、ラップ、距離は、Apple Watchに内蔵されている加速度センサーを使って記録されます。  
![丸に囲まれたチェックマークは正しい使い方を示しています。](https://docs-assets.developer.apple.com/published/88662da92338267bb64cd2275c84e484/checkmark%402x.png)| このタイプのワークアウトでは、センサーによる読み取りが使用できないときは、早歩きの場合と同等のカロリーを消費したものとみなされます。  
![丸に囲まれたチェックマークは正しい使い方を示しています。](https://docs-assets.developer.apple.com/published/88662da92338267bb64cd2275c84e484/checkmark%402x.png)| 自由形で泳ぐ場合は、GPSで距離だけが提示されます。水中では心拍数を計測できない場合がありますが、カロリーは内蔵の加速度センサーを使って記録されます。  
  
**セッションの終了時に概要を表示する。** 概要画面では、ワークアウトが完了したことの確認と記録された情報の表示を行います。ユーザが現在の進行状況を簡単に確認できるように、アクティビティリングを追加して、概要をより充実させることを検討してください。

**極端に短いワークアウトセッションは破棄する。** セッションが開始してから数秒で終了した場合は、データを自動的に破棄するか、データをワークアウトとして記録するかどうかの選択をユーザに求めます。

**ユーザが動いていてもテキストを読めるようにする。** 体を動かすセッションでは、大きなフォントサイズを使用し、最も重要な情報が読みやすくなるようにテキストを配置します。

**アクティビティリングを正しく使う。** アクティビティリングビューは、アクティビティアプリと同じ色と意味を持つ1つ以上のリングからなるAppleデザインの要素です。このビューは記載されている目的でのみ使用してください。

## [プラットフォームの考慮事項](/jp/design/human-interface-guidelines/workouts#Platform-considerations)

 _iOS、iPadOS、watchOSに追加の考慮事項はありません。macOS、tvOS、visionOSには対応していません。_
