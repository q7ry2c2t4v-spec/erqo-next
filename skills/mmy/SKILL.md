# /mmy - セッションログ記録スキル

## 概要

現在のセッション内容をレポートにまとめ、.libs/session-logs/ に保存する。

## 実行手順

### 1. セッション内容の分析

会話の全体を振り返り、以下を整理する:
- 何をしたか（作業内容）
- 何を決めたか（決定事項）
- やり残し・次にやるべきこと（未完了）

### 2. 識別コードの決定

`SLOG-YYYYMMDD-NNN` 形式で識別コードを生成する。

- `YYYYMMDD` — 今日の日付
- `NNN` — その日の連番（001 から開始）

連番の決定方法:
1. `.libs/session-logs/YYYY-MM/` 内の既存ファイルを確認
2. 同日の `slog-YYYYMMDD-*.md` の最大番号 + 1

### 3. レポートの作成

**保存先:** `.libs/session-logs/YYYY-MM/slog-YYYYMMDD-NNN.md`

```markdown
# SLOG-YYYYMMDD-NNN — セッションタイトル

関連: 関連ページの識別コード
タグ: 機能タグ, 作業種別タグ, feature-tag, work-type-tag

## SLOG-YYYYMMDD-NNN.概要 — 概要
セッションの要約（何をしたか）

## SLOG-YYYYMMDD-NNN.作業内容 — 作業内容
具体的にやったことの詳細

## SLOG-YYYYMMDD-NNN.決定事項 — 決定事項
セッション中に決めたこと

## SLOG-YYYYMMDD-NNN.未完了 — 未完了・次回
やり残し、次にやるべきこと
```

### タグのルール

タグは 2 種類の役割を兼ねる:

| 役割 | 例 |
|---|---|
| 機能・トピック | 認証, JWT, DB設計, API, auth, database |
| 作業の種類 | 実装, 設計, バグ修正, リファクタ, implementation, design |

日本語タグと英語タグの両方を付与する。

### 4. 品質セルフチェック

保存前に以下の基準でセルフチェックを行う:

1. **復元可能性** — このログだけ読んで次回再開できるか？
2. **決定事項の具体性** — 理由と結論が明記されているか？
3. **未完了の実行可能性** — 次にやることが具体的で着手可能か？

基準を満たさない場合はログを改善してから保存する。

### 5. インデックス更新

レポート作成後、以下を実行する:

```bash
python "{nxt}/core/index.py" reindex
```

### 重要なルール

1. **復元可能性** — セッションの知識を次回以降に引き継げるだけの詳細さを保つ
2. **170行目安** — 1ファイル170行前後を目安とする
3. **関連ページ** — セッション中に触れた既存ページがあれば `関連:` に記載する（片方向でよい）
4. **セッションタイトル** — 作業の主題が一目でわかる簡潔なタイトルを付ける
5. **品質セルフチェック** — 保存前に3つの基準を満たしているか確認する
