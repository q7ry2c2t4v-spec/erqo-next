# /gh — プロジェクト Git 管理スキル

## 概要

プロジェクトの git コミット・プッシュを安全に実行する。

## いつ使うか

- 変更をコミットしたいとき
- リモートにプッシュしたいとき
- 現在の差分を確認したいとき

## 実行手順

### 差分確認

```bash
python "{nxt}/skills/gh/handler.py" check
```

ブランチ名・変更ファイル一覧・差分統計を表示する。

### コミット

```bash
python "{nxt}/skills/gh/handler.py" commit "コミットメッセージ"
```

`git add -A` で全変更をステージングし、コミットする。

### プッシュ

```bash
python "{nxt}/skills/gh/handler.py" push
```

現在のブランチをリモート (origin) にプッシュする。

### コミット + プッシュ (一括)

```bash
python "{nxt}/skills/gh/handler.py" full "コミットメッセージ"
```

コミットとプッシュを一括で実行する。

## 重要なルール

1. **コミット前に差分確認** — `check` で変更内容を把握してからコミットする
2. **コミットメッセージ** — `feat(ID): 内容` の形式で書く
3. **プッシュの確認** — プッシュは必ずユーザーの承認を得てから実行する
4. **変更なしの場合** — 変更がなければコミットしない (自動スキップ)
