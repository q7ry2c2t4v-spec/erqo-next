# /ruru - プロジェクトルール管理スキル

## 概要

プロジェクトルール（スタイル・構成方針等）を登録・更新する。開発フローとは独立した単独スキル。

## 実行手順

### 1. ユーザーとの対話

ルールの内容をユーザーと対話して決める:
- どんなルールか
- 適用範囲（コード全体 / 特定領域）
- 具体的な規約

### 2. スコープの自動推定

ルール内容からスコープ（適用範囲）を自動推定する:

| 内容のキーワード | 推定スコープ |
|---|---|
| コンポーネント, レイアウト, 画面, UI, CSS | `適用: ui` |
| API, エンドポイント, データベース | `適用: api` |
| 上記に該当しない / 全般 | `適用: all` |

推定結果をユーザーに確認してから適用する。

### 3. ルールファイルの作成・更新

承認を取ってから作成する。

**保存先:** `.libs/rules/`

ファイル名はルールの内容を表す kebab-case + `.md`

```markdown
# RULE-STYLE — コーディングスタイルルール

適用: ui
タグ: ルール, スタイル, rule, style

## RULE-STYLE.概要 — 概要
（必須。load.py の閾値超過時はここだけロードされる）

## RULE-STYLE.詳細 — 詳細
（ルール本文）
```

**`適用:` メタデータ** はヘッダー直後 (タグの前) に記載する。load.py がスコープフィルタリングに使用する。

### 4. CLAUDE.md への @import 追加（新規ファイル時のみ）

新しいルールファイルを作成した場合、CLAUDE.md に `@import` 行を追加する:

```markdown
## プロジェクトルール
@.libs/rules/style.md
```

- CLAUDE.md にルール本体をハードコーディングしない
- `@` 参照行のみ記載する
- 既存ファイルの更新時は CLAUDE.md の変更は不要

### 5. インデックス更新

作成後、以下を実行する:

```bash
python "{nxt}/core/index.py" reindex
```

### 重要なルール

1. **ルール本体は rules/ で管理** — CLAUDE.md には参照のみ
2. **`適用:` メタデータ必須** — スコープ自動推定の結果をユーザーに確認して記載する
3. **ページフォーマット準拠** — ヘッダー、タグ行、セクション記法を守る
4. **170行目安** — 情報量が多い場合はファイルを分割する
5. **既存ルールの確認** — 新規作成前に `.libs/rules/` の既存ファイルを確認し、重複や矛盾がないか確認する
