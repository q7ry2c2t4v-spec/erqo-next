# /rsrc - ウェブリサーチ → .libs 保存

## 概要

指定されたトピック `$ARGUMENTS` についてウェブリサーチを行い、結果を `.libs/research/` にページとして保存する。
識別コード・タグ付きで保存されるため、`index.py collect` のキーワード検索で `/tp` から自動参照される。

## 実行手順

### 1. トピックの整理

ユーザーと対話してリサーチの方針を決める:
- リサーチの目的・範囲
- 知りたい具体的な事柄
- 深さ（概要レベル / 実装レベル）

識別コードの命名を決定する:

**形式:** `RSRC-<TOPIC>-<SUBTOPIC>`
- 例: `RSRC-OAUTH-FLOW`, `RSRC-WEBSOCKET-PROTOCOL`
- カテゴリは常に `RSRC`

### 2. ウェブリサーチ

承認を取ってからリサーチを実行する。

- WebSearch で広く情報を収集する
- WebFetch で有用なページの詳細を取得する
- 複数の情報源を確認し、正確性を担保する
- 公式ドキュメント・仕様書を優先する

### 3. リサーチページの作成

**保存先:** `.libs/research/<topic>/`

`<topic>` は識別コードの2番目のセグメントを小文字にしたもの（例: `RSRC-OAUTH-FLOW` → `research/oauth/`）

**ファイル名:** 識別コードを小文字にしたもの + `.md`

**ページフォーマット:**

```markdown
# RSRC-OAUTH-FLOW — OAuth 2.0 認証フロー

関連: （関連ページの識別コードがあれば記載）
タグ: OAuth, 認証, auth, authorization, flow

## RSRC-OAUTH-FLOW.概要 — 概要
（何を調べたか、リサーチの目的）

## RSRC-OAUTH-FLOW.調査結果 — 調査結果
（主要な発見・結論の要約）

## RSRC-OAUTH-FLOW.技術詳細 — 技術詳細
（具体的な仕様・仕組み・動作フロー）

## RSRC-OAUTH-FLOW.比較 — 比較・選択肢
（代替案がある場合の比較。該当しなければ「該当なし」と書く）

## RSRC-OAUTH-FLOW.適用 — プロジェクトへの適用
（このプロジェクトでどう活用できるか）

## RSRC-OAUTH-FLOW.出典 — 出典
（情報源の URL・名前・参照日）
```

### 4. インデックス更新

作成後、以下を実行する:

```bash
python "{nxt}/core/index.py" reindex
```

### 5. リンク案内

既にプロジェクトに関連する TP ページや features ページがある場合:
- そのページの `関連:` にリサーチページの識別コードを追加することを提案する

## 重要なルール

1. **ページフォーマット準拠** — ヘッダー、タグ行、関連行、セクション記法を守る
2. **170行目安** — 超える場合はページを分割する
3. **タグの充実** — index.py collect で発見されるよう、関連キーワードを幅広く設定する
4. **出典の明記** — 情報源を必ず記録する。参照日も書く
5. **正確性** — 複数の情報源を突き合わせ、古い情報や誤りに注意する
6. **復元可能性** — この記録だけで知識を再構築できるレベルの詳細さを保つ
