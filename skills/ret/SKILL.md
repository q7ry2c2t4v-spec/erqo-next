# /ret - セッション復元スキル

## 概要

直近のセッションログを取得し、前回の作業コンテキストを復元する。

## いつ使うか

- 新しいセッションの最初に、前回の続きをやりたいとき
- ファーストターンの内容からキーワードが識別できないとき
- 「前回何やってたっけ？」を素早く思い出したいとき

## 実行手順

### 1. 直近セッションログの取得

`.libs/session-logs/` 配下から直近のセッションログを探す:

1. 最新の月フォルダ（`YYYY-MM/`）を特定する
2. その中の最新の `slog-*.md` ファイルを読み込む
3. 必要に応じて直近2〜3件を読む

### 2. コンテキストの要約

取得したセッションログから以下を整理して伝える:
- 前回のセッションで何をしたか
- 決定事項
- やり残し・次にやるべきこと

### 3. 関連ページの自動収集

セッションログのタグと関連ページから、キーワードを抽出して自動で関連ページを収集する:

```bash
python "{nxt}/core/index.py" collect キーワード1 キーワード2
```

収集されたページの一覧を提示し、必要に応じて内容を読み込む。

### 重要なルール

1. **読むだけ** — /ret はファイルを読んで情報を伝えるだけ。ファイルの作成・更新はしない
2. **簡潔に** — 前回のコンテキストを把握できる最小限の情報を伝える。全文を読み上げない
3. **次のアクションを提案** — 未完了事項があれば、次に何をすべきか提案する
4. **関連ページの自動収集** — セッションログのタグから index.py collect を実行し、関連コンテキストを拡充する
